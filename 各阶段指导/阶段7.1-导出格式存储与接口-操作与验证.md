# 阶段 7.1 导出格式存储与接口 — 操作与验证指南

本文档面向执行与验收，按「阶段要达成什么 → 需要准备什么 → 如何验证」说明 **开发计划** 阶段 7.1（导出格式存储与接口）的完整流程。阶段 7.1 在后端新增「导出格式」配置的存储（单行表 `export_format_settings`）与 GET/POST `/api/settings/export-format` 接口，未配置时返回与当前 4.3 一致的默认值，为阶段 7.2/7.3 提供数据与 API 基础。

---

## 一、阶段 7.1 要达成什么

- **存储**：新增表 `export_format_settings`，单行维护；字段含一级/二/三级标题字体与字号、正文字体字号、表格内字体字号、首行缩进（磅）、行距（倍数）。
- **默认值**：无行或列为空时，GET 与 7.2 的 `format_options` 使用同一套默认（如一级标题宋体 22pt、正文 12pt、首行缩进 24pt、行距 1.5）。
- **接口**：`GET /api/settings/export-format` 返回当前配置（扁平 JSON）；`POST /api/settings/export-format` 接受可选字段、校验字号 8–72、首行缩进 0–100、行距 0.5–3.0，保存后返回完整配置。
- **交付物**：表与模型、`get_export_format_config()` / `set_export_format_config()`、GET/POST 接口；7.2 可直接使用 `get_export_format_config()` 作为 `format_options`。

## 三、涉及文件与结构

- **模型**：`backend/app/models/export_format_setting.py`，表 `export_format_settings`，列：id、heading_1_font、heading_1_size_pt、…、first_line_indent_pt、line_spacing。
- **存储层**：`backend/app/settings_store.py` 中 `DEFAULT_EXPORT_FORMAT`、`get_export_format_config()`、`set_export_format_config(...)`。
- **接口**：`backend/app/routers/settings.py` 中 `GET /export-format`、`POST /export-format`（Pydantic 体 `PostExportFormatBody`，全部可选）。

## 四、如何验证阶段 7.1

### 1. 启动后端

按 [开发前-服务启动顺序.md](开发前-服务启动顺序.md) 启动后端（uvicorn）；确认无报错，启动时表 `export_format_settings` 会随 `Base.metadata.create_all` 创建。

### 2. 验证 GET 默认配置

- 浏览器打开：`http://localhost:8000/api/settings/export-format`
- 或 Swagger：`http://localhost:8000/docs` → GET `/api/settings/export-format` → Try it out → Execute

**预期**：返回 200，JSON 含 12 个键，例如：`heading_1_font`、`heading_1_size_pt`、…、`body_font`、`body_size_pt`、`table_font`、`table_size_pt`、`first_line_indent_pt`、`line_spacing`；值为默认（如宋体、22/16/14/12、24、1.5）。

### 3. 验证 POST 保存与再 GET

- 在 Swagger 中：POST `/api/settings/export-format`，Request body 填例如：`{"body_font":"黑体","body_size_pt":14,"line_spacing":1.25}`
- 执行后应返回 200，响应为当前完整配置（其中 body_font、body_size_pt、line_spacing 为刚传的值）。
- 再执行 GET `/api/settings/export-format`，应得到与 POST 响应一致的配置。

### 4. 验证校验（非法值返回 400）

- POST body 为 `{"body_size_pt":200}`：预期 400，提示字号应在 8–72 之间。
- POST body 为 `{"line_spacing":5.0}`：预期 400，提示行距应在 0.5–3.0 之间。

### 5. 验收标准汇总

| 项 | 标准 |
|----|------|
| 表与启动 | 后端启动无报错，表 `export_format_settings` 存在 |
| GET 默认 | 无配置时 GET 返回 12 项默认值（宋体、22/16/14/12、24、1.5 等） |
| POST 与再 GET | POST 合法 body 后返回完整配置；再 GET 与之一致 |
| 校验 | 字号超 72、行距超 3.0 等返回 400 及明确提示 |

## 五、与 7.2、7.3 的衔接

- **阶段 7.2**：下载接口调用 `get_export_format_config()`，将返回值传入 `markdown_to_docx(md, format_options)`；`export_docx.py` 根据 `format_options` 应用标题/正文/表格字体字号、首行缩进、行距。
- **阶段 7.3**：设置页「导出格式」区块 GET `/api/settings/export-format` 回填表单，提交时 POST `/api/settings/export-format`；字段与 API 扁平键一一对应。
