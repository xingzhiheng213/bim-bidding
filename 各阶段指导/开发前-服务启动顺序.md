# 开发前 — 服务启动顺序（本机安装版）

本文档说明**下次开发前**按什么顺序启动各项服务，避免遗漏。适用于 **Redis、PostgreSQL、后端、前端均为本机安装**（未使用 Docker）的情况。

---

## 一、需要开哪些、开几个终端

| 顺序 | 服务 | 何时需要 | 占用的终端 |
|------|------|----------|------------|
| 0 | Redis、PostgreSQL | 后端要连 DB、Celery 要连 Redis；本机安装时多为系统服务，不一定单独开终端 | 见下文「二」 |
| 1 | 后端 FastAPI（uvicorn） | 只要用前端或调 API 就要开 | **终端 1** |
| 2 | Celery Worker | 阶段 1.2 起（文档解析、LLM 等异步任务）才需要 | **终端 2**（阶段 1.2 起） |
| 3 | 前端（npm run dev） | 要用浏览器访问页面就要开 | **终端 3**（或 2，若未开 Celery） |

**当前阶段（0.2 / 0.3 / 1.1）**：只需 **终端 1（后端）+ 终端 2（前端）**，共 2 个终端。  
**阶段 1.2 起**（文档解析、LLM 等 Celery 任务）：需要 **终端 1（后端）+ 终端 2（Celery）+ 终端 3（前端）**，共 3 个终端。

---

## 二、Redis 与 PostgreSQL（本机安装、未用 Docker）

两者均为**本机安装**时，通常以 Windows 服务形式存在，不一定每次手动开。

### 1. 确认是否已在运行

- **Redis**：端口 6379。可在 PowerShell 执行 `redis-cli ping`，若返回 `PONG` 表示已运行；若提示“无法连接”则需启动。
- **PostgreSQL**：端口 5432。可打开 **pgAdmin** 或 **服务**（`services.msc`）中查看是否有名为 `postgresql-x64-xx` 的服务并处于“正在运行”。

### 2. 若未运行，如何启动

- **Redis**  
  - 若安装时注册为 Windows 服务：在“服务”里找到 Redis，右键“启动”。  
  - 或从开始菜单/安装目录运行 **Redis 服务器**（如 `redis-server.exe`），会占一个窗口，不要关。
- **PostgreSQL**  
  - 在“服务”里找到 PostgreSQL 服务，右键“启动”。  
  - 或通过 **pgAdmin** 连接本机，若连得上说明已启动。

**建议**：若经常开发，可将 Redis、PostgreSQL 设为“自动”启动，这样开机后无需每次手动开。

## 三、按顺序启动（每次开发前）

项目路径假设为 `d:\标书工作流`，若你的路径不同请替换。

### 第 0 步：确认 Redis、PostgreSQL 已运行（本机）

- 按上面「二」检查；若未运行则先启动，再继续下面步骤。

---

### 第 1 步：启动后端 FastAPI（终端 1）

1. 打开 **CMD 或 PowerShell**。
2. 执行：

```powershell
cd d:\标书工作流\backend
.venv\Scripts\activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8001
```

3. 看到 `Uvicorn running on http://0.0.0.0:8001` 和 `Application startup complete.` 即表示成功。**此窗口不要关**。

---

### 第 2 步：启动 Celery Worker（终端 2）— 阶段 1.2 起才需要

若当前只做到 0.2 / 0.3 / 1.1，可**跳过**本步；做到阶段 1.2（文档解析等）时**必须**开。阶段 1.2 的完整操作与验证见 [阶段1.2-文档解析-操作与验证.md](阶段1.2-文档解析-操作与验证.md)。

1. **再开一个** CMD 或 PowerShell。
2. 执行：

```powershell
cd d:\标书工作流\backend
.venv\Scripts\activate
celery -A celery_app worker -l info -P solo
```

3. 看到 `celery@xxx ready.` 即表示成功。**此窗口不要关**。

---

### 第 3 步：启动前端（再开一个终端）

阶段 1.3（前端上传与状态）的完整操作与验证见 [阶段1.3-前端上传与状态-操作与验证.md](阶段1.3-前端上传与状态-操作与验证.md)。

1. **再开一个** CMD 或 PowerShell。
2. 执行：

```powershell
cd d:\标书工作流\frontend
npm run dev
```

3. 看到 `Local: http://localhost:5173/` 即表示成功。
4. 浏览器打开 <http://localhost:5173> 即可开发。**此窗口不要关**。

---

## 四、快速自检

- **后端**：浏览器打开 http://localhost:8001/health ，应显示 `{"status":"ok"}`。
- **前端**：浏览器打开 http://localhost:5173 ，应看到「BIM 标书生成」和「后端状态：ok」。
- **Celery**（阶段 1.2 起）：若 Worker 终端里能看到 `ready` 且无报错，即正常。

---

## 五、小结：下次开发前要做的事（本机版）

1. 确认 **Redis**、**PostgreSQL** 已运行（本机服务或手动启动，见「二」）。
2. **终端 1**：`cd backend` → 激活 venv → `uvicorn app.main:app --reload --host 0.0.0.0 --port 8001`。
3. **终端 2**（阶段 1.2 起）：`cd backend` → 激活 venv → `celery -A celery_app worker -l info -P solo`。
4. **终端 3**（或 2）：`cd frontend` → `npm run dev`。
5. 浏览器访问 http://localhost:5173 开始开发；开发期间上述终端都不要关。

开发结束后直接关闭这些终端即可；下次开发时按上述顺序重新开一遍。
