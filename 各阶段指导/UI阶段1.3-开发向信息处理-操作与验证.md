# UI 阶段 1.3 开发向信息处理 — 操作与验证指南

本文档面向执行与验收，按「阶段要达成什么 → 需要准备什么 → 如何验证」说明 **UI 设计计划** 阶段 1.3（开发向信息处理）的完整流程。阶段 1.3 在 1.1/1.2 侧栏布局与导航基础上，**移除或隐藏**首页「后端状态：ok」等开发用文案；检查其他页面并弱化冗余标题，保持信息简洁。若需保留自检能力，可采用「仅开发环境显示」或「移至设置页调试区块」。

---

## 一、阶段 1.3 要达成什么

- **首页**：移除或隐藏「后端状态：ok」及承载该文案的区块；若需保留，可收到设置页「调试」区块或仅开发环境（如 `import.meta.env.DEV`）显示。与 health 请求相关的「后端请求失败」等错误态一并移除或仅在约定条件下显示。
- **其他页面**：检查并移除仅面向开发的提示或冗余标题（如内容区重复大标题）；保持信息简洁。设置页、对比页、任务详情页的主标题（设置、文本对比、任务详情）保留；错误态文案可保留或改为通用表述。
- **交付物**：首页已移除或隐藏「后端状态」等开发用文案（或仅开发环境/设置页调试区块展示）；其他页面无新增开发向信息，冗余标题已按需弱化。
- **验收标准**：全站侧栏 + 主内容区布局与导航、侧栏高亮正确；**开发向信息已移除或可配置隐藏**。

## 三、阶段 1.3 交付物说明

完成 1.3 后，应满足以下内容：

### 1. 首页：开发用文案已移除或可配置隐藏

- **方案 A（推荐）**：直接移除首页对 `/health` 的请求与展示；不再显示「后端状态：ok」及「后端请求失败」区块。首页仅保留：主标题（可弱化为「我的任务」等，避免与侧栏品牌重复）、创建任务按钮、任务列表。
- **方案 B**：保留 `getHealth` 请求，但仅在开发环境（`import.meta.env.DEV`）下渲染「后端状态：…」区块；生产构建下不渲染、可不请求或仍请求但不展示；错误态也仅在开发环境显示。
- **方案 C**：在设置页增加「调试」区块（可折叠），仅开发环境显示或始终显示；将「后端状态」展示移至该区块，首页不再展示；需在 SettingsPage 中请求 health 并展示。

任选其一即可；计划表述为「移除或隐藏」「可收到设置页调试区块或仅开发环境显示」。

### 2. 首页：冗余标题弱化（可选）

- 若首页内容区主标题与侧栏品牌「BIM 标书生成」重复，可弱化或改为页面含义更明确的标题（如「我的任务」），或删除该主标题仅保留「任务列表」小标题与创建任务按钮，以保持信息简洁。

### 3. 其他页面检查

- **SettingsPage**：主标题「设置」、区块标题「大模型 API」「模型配置」等保留；错误文案「无法连接后端…」可保留（面向用户）或改为「无法连接服务，请检查网络与配置」。
- **ComparePage**：主标题「文本对比（标红删除、标绿新增）」保留。
- **TaskDetailPage**：主标题「任务详情」保留。
- 无其他明显「仅面向开发」的文案需在本阶段移除；若有遗漏，在走查时补充。

## 四、如何验证阶段 1.3

### 1. 验证「首页无开发向文案」

1. 启动前端：`cd d:\标书工作流\frontend`，`npm run dev`。
2. 打开 http://localhost:5173/ 。
3. **首屏**不应出现「后端状态：ok」或「后端请求失败」等开发用文案（若采用方案 B 或 C，则仅在约定条件下显示，如开发环境或设置页调试区块）。
4. 首页应显示主标题（如「我的任务」）、创建任务按钮、任务列表；信息简洁无重复品牌大标题（若已弱化）。

### 2. 验证「其他页面无新增开发向信息」

- 打开设置页、对比页、任务详情页：主标题与区块标题保留，无「仅面向开发」的提示；错误态文案可保留或为通用表述。

### 3. 验收标准汇总

| 项 | 标准 |
|----|------|
| 首页 | 「后端状态：ok」等开发用文案已移除或仅在约定条件下显示（开发环境/设置页调试）；首页首屏信息简洁 |
| 冗余标题 | 首页内容区主标题已按需弱化（如改为「我的任务」），与侧栏品牌不重复堆砌 |
| 其他页面 | 无新增开发向文案；主标题与错误提示符合对外产品界面 |

## 五、与 1.2、阶段 2 的衔接

- **阶段 1.2**：1.2 完成侧栏导航与高亮；1.3 在相同布局下移除或隐藏开发向信息，不改变侧栏与路由。
- **阶段 2**：阶段 1 验收标准「开发向信息已移除或可配置隐藏」满足后，进入阶段 2（任务详情页流程可视化）；1.3 不涉及任务详情页步骤条与卡片。

若采用方案 B 或 C，建议在 README 或本指南中注明「后端状态」的显示条件（如仅 `npm run dev` 时或仅在设置页调试区块），便于后续维护。
