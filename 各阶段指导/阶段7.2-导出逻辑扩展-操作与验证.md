# 阶段 7.2 导出逻辑扩展 — 操作与验证指南

本文档面向执行与验收，按「阶段要达成什么 → 需要准备什么 → 如何验证」说明 **开发计划** 阶段 7.2（导出逻辑扩展）的完整流程。阶段 7.2 扩展 `markdown_to_docx` 支持 `format_options` 参数，对标题/正文/表格应用字体字号、首行缩进、行距；下载接口在生成 DOCX 前读取导出格式配置并传入 `markdown_to_docx`，使下载的 Word 应用用户设置。

---

## 一、阶段 7.2 要达成什么

- **export_docx**：`markdown_to_docx(markdown_text, format_options=None)` 支持传入格式配置；根据 `format_options` 对 Heading 1/2/3、正文（含列表）、表格单元格内的段落应用字体、字号、首行缩进、行距；未传或某键缺失时与 `DEFAULT_EXPORT_FORMAT` 合并后使用。
- **下载接口**：`GET /api/tasks/{task_id}/download` 在生成 DOCX 前调用 `get_export_format_config()`，将返回值传入 `markdown_to_docx(md, format_options)`，使下载文件应用用户设置的样式。
- **交付物**：`export_docx.py` 中扩展签名与 `_apply_heading_style`、`_apply_body_style`、`_apply_table_cell_style`；`tasks.py` 中 download_docx 读取配置并传入 `markdown_to_docx`。

## 三、涉及文件与结构

- **export_docx**：`backend/app/export_docx.py` 中 `markdown_to_docx(markdown_text, format_options=None)`；合并 `opts = {**DEFAULT_EXPORT_FORMAT, **(format_options or {})}`；`_apply_heading_style(paragraph, level, opts)`、`_apply_body_style(paragraph, opts)`、`_apply_table_cell_style(cell, opts)`；在每次 add_heading / add_paragraph / 表格 cell 赋值后调用上述辅助函数。
- **下载**：`backend/app/routers/tasks.py` 中 `download_docx` 内 `format_options = get_export_format_config()`，`doc = markdown_to_docx(md, format_options)`。

## 四、如何验证阶段 7.2

### 1. 通过 API 保存导出格式

- 在 Swagger 或 Postman 中 POST `/api/settings/export-format`，body 例如：`{"body_font":"黑体","body_size_pt":14,"line_spacing":1.5,"first_line_indent_pt":24}`，确认返回 200。

### 2. 下载 DOCX 并检查样式

- 进入一个已有按章生成内容的任务详情，点击「下载 Word」。
- 打开下载的 DOCX：正文段落应为黑体 14pt、首行缩进约两字符、行距 1.5 倍；一级/二级/三级标题为配置的字体字号（未改则仍为宋体 22/16/14）；表格内文字为配置的表格字体字号（未改则宋体 12pt）。

### 3. 未配置时行为

- 若不调用 POST export-format（或清空配置后不保存），下载的 DOCX 应使用默认样式（与阶段 4.3 一致）：宋体、22/16/14/12、首行缩进 24pt、行距 1.5。

### 4. 验收标准汇总

| 项 | 标准 |
|----|------|
| 配置生效 | 在设置中保存导出格式后，下载的 DOCX 中标题/正文/表格的字体字号、首行缩进、行距与配置一致 |
| 未配置 | 未配置时下载的 DOCX 与 4.3 行为一致（默认宋体、22/16/14/12、首行缩进 24、行距 1.5） |
| 结构不变 | 不改变现有 Markdown 解析与 DOCX 结构（标题/段落/列表/表格），仅叠加样式 |

## 五、与 7.1、7.3 的衔接

- **阶段 7.1**：`get_export_format_config()` 返回的 dict 直接作为 `format_options` 传入 `markdown_to_docx`；7.2 与 7.1 约定一致。
- **阶段 7.3**：前端设置页「导出格式」表单保存后，用户下载 DOCX 即使用刚保存的配置；7.2 无需再改后端。
