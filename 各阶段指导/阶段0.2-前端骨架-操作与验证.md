# 阶段 0.2 前端骨架 — 操作与验证指南（新手向）

本文档面向新手，按「需要准备什么 → 一步步操作 → 如何验证」说明阶段 0.2 的完整流程。阶段 0.2 只做**前端骨架**（Vite + React + TypeScript + Ant Design），并实现与后端的健康检查联调。

---

## 一、阶段 0.2 要达成什么

- 在电脑上能**启动前端开发服务器**（Vite），用浏览器打开页面。
- 页面有**最简布局**：顶部导航 + 主内容区，主内容区显示「BIM 标书生成」标题。
- 页面能**请求后端**的 `/health` 接口，并显示返回结果（如「后端状态：ok」）。
- 浏览器控制台**无 CORS 跨域错误**（需后端已启用 CORS、且后端先启动）。

---

## 三、环境准备（Node.js）

### Node.js

- 若未安装，请从 [Node.js 官网](https://nodejs.org/) 下载 LTS 版本安装（通常已包含 npm）。
- **验证**：在终端执行 `node --version`（应 ≥ 18）、`npm --version`，有输出即可。

前端不需要单独安装 Redis 或 PostgreSQL，只要本机有 Node.js；**请求后端**时由后端所在机器提供 API（本地开发时即本机已启动的后端）。

## 四、前端操作步骤（按顺序做）

### 步骤 1：进入前端目录

在终端（PowerShell 或 CMD）中执行：

```powershell
cd d:\标书工作流\frontend
```

（若你的项目不在 D 盘，改成你的实际路径。）

---

### 步骤 2：安装依赖

```powershell
npm install
```

等待安装完成，无报错即可。会生成 `node_modules/` 目录。

---

### 步骤 3：配置环境变量（可选）

前端通过环境变量 `VITE_API_BASE` 指定后端 API 根地址。项目里已提供 `frontend/.env.development`，默认内容为：

```env
VITE_API_BASE=http://localhost:8001
```

- 若你的后端跑在本机且端口为 8001，**无需修改**。
- 若后端端口或主机不同（例如端口 3000），可编辑 `.env.development` 或新建 `.env.local` 覆盖，例如：  
  `VITE_API_BASE=http://localhost:3000`

保存后，下次执行 `npm run dev` 时会生效。

---

### 步骤 4：启动后端（第一个终端）

前端会请求后端的 `/health`，因此**必须先启动后端**。若后端已在运行可跳过。

在**第一个终端**中执行：

```powershell
cd d:\标书工作流\backend
.venv\Scripts\activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8001
```

看到 `Uvicorn running on http://0.0.0.0:8001` 和 `Application startup complete.` 即表示后端已就绪。**不要关闭此终端**。

---

### 步骤 5：启动前端（第二个终端）

再开一个终端窗口，执行：

```powershell
cd d:\标书工作流\frontend
npm run dev
```

看到类似 `VITE ready in xxx ms` 和 `Local: http://localhost:5173/` 即表示前端开发服务器已启动。

---

### 步骤 6：浏览器验证

1. 浏览器打开：<http://localhost:5173>
2. 页面上应看到：
   - 顶部导航栏有「BIM 标书生成」；
   - 主内容区有标题「BIM 标书生成」；
   - 下方有「后端状态：**ok**」（表示已成功请求后端 `/health`）。
3. 按 F12 打开开发者工具，切到「控制台」或「Console」，应**没有**红色的 CORS 跨域报错。

若页面显示「后端请求失败」或网络错误：请确认**后端终端**（步骤 4）仍在运行，且地址为 `http://localhost:8001`；若你改了端口，需同步修改前端的 `VITE_API_BASE`。

## 五、验证（阶段 0.2 验收）

按下面三项逐项检查，都通过即表示阶段 0.2 完成。

### 验证 1：前端能启动

1. 在 `frontend` 目录下执行 `npm run dev`。
2. 终端中应出现 `Local: http://localhost:5173/`（或类似），无报错退出。

若报错「找不到模块」等：先在 `frontend` 下执行一次 `npm install`。

---

### 验证 2：页面展示标题与健康检查结果

1. 保持**后端**（uvicorn）和**前端**（npm run dev）都在运行。
2. 浏览器打开 http://localhost:5173 。
3. 页面应显示：
   - 「BIM 标书生成」标题（顶部导航 + 主内容区各一处）；
   - 「后端状态：**ok**」（或等价展示 `/health` 返回的 `status`）。

若显示「后端请求失败」：检查后端是否在 8001 端口运行；检查 `frontend/.env.development` 或 `.env.local` 中 `VITE_API_BASE` 是否与后端地址一致。

---

### 验证 3：无 CORS 跨域错误

1. 在浏览器中打开 http://localhost:5173（且页面已成功显示「后端状态：ok」）。
2. 按 F12 打开开发者工具，切换到「Console」。
3. 控制台中不应出现包含 `CORS`、`Access-Control-Allow-Origin` 等字样的红色报错。

若出现 CORS 报错：说明后端未正确启用 CORS。请确认后端使用的是已加入 CORS 中间件的版本（见 [README-backend.md](../../README-backend.md)）；后端默认允许 `http://localhost:5173` 和 `http://127.0.0.1:5173`。

## 六、常见问题

| 现象 | 可能原因 | 处理办法 |
|------|----------|----------|
| 页面显示「后端请求失败」或网络错误 | 后端未启动，或端口/主机不对 | 先在 backend 下启动 uvicorn（端口 8001）；检查 `VITE_API_BASE` 是否为 `http://localhost:8001` |
| 控制台报 CORS 错误 | 后端未挂载 CORS 中间件 | 确认 backend 的 `app/main.py` 中已添加 `CORSMiddleware`，并重启 uvicorn |
| `npm install` 很慢或失败 | 网络或镜像问题 | 可尝试切换 npm 镜像：`npm config set registry https://registry.npmmirror.com` 后再执行 `npm install` |
| 修改 `.env.development` 后不生效 | Vite 需重启才读环境变量 | 关掉 `npm run dev` 后重新执行 `npm run dev` |
| 端口 5173 被占用 | 本机其他程序占用了 5173 | 可修改 `frontend/vite.config.ts` 里 `server.port` 为其他端口（如 5174），再访问新端口 |
