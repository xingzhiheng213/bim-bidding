# UI 阶段 2.2 按步骤分块展示 — 操作与验证指南

本文档面向执行与验收，按「阶段要达成什么 → 需要准备什么 → 如何验证」说明 **UI 设计计划** 阶段 2.2（按步骤分块展示）的完整流程。阶段 2.2 将任务详情页内容重构为「每步一卡片」：7 个步骤各对应一个 Ant Design Card，卡片内含步骤名、状态 Tag、简短说明、该步主操作按钮；详情内容（解析结果、分析文本、框架列表、章节预览等）放入卡片内 Collapse，默认收起；按钮仅在该步可操作时展示，不可用时有一句明确说明。

---

## 一、阶段 2.2 要达成什么

- **每步一卡片**：与步骤条一致，按固定顺序（上传 → 解析 → 分析 → 参数 → 框架 → 按章生成 → 导出）渲染 **7 张 Card**；每张 Card 保留 `id="step-{step_key}"`，便于 2.1 步骤条点击滚动。
- **卡片头部**：步骤名称（STEP_TITLES）、右侧或标题旁为**状态 Tag**（stepStatusDisplay 的 tagColor 与 getStepStatusLabel）。
- **卡片正文**：简短说明（该步作用或当前状态）；该步主操作按钮**仅在该步可操作时展示**（如解析完成才显示「开始分析」）；不可操作时展示一句说明（如「请先完成上传」「请先完成解析」）。
- **详情可折叠**：解析预览、分析结果、参数项目信息/BIM/风险点、框架章节列表、按章章节列表等长内容均放在卡片内的 **Collapse**，**默认收起**（defaultActiveKey=[]），避免一屏信息过载。
- **交付物**：任务详情页按步骤分块；操作入口清晰；长内容可折叠。

## 三、阶段 2.2 交付物说明

完成 2.2 后，应满足以下内容：

### 1. 统一卡片结构（`frontend/src/pages/TaskDetailPage.tsx`）

- **7 张卡片**：`TASK_STEP_ORDER.map(stepKey => <Card key={stepKey} id={"step-"+stepKey} ...>)`，顺序固定为 upload → extract → analyze → params → framework → chapters → export。
- **Card 标题**：`title={STEP_TITLES[stepKey]}`。
- **Card 右侧**：`extra={<Tag color={stepStatusDisplay[status].tagColor}>{getStepStatusLabel(status)}</Tag>}`，status 来自该 step_key 在 data.steps 中的步骤，不存在则为 pending。
- **卡片间距**：Card 的 style 或容器使用 `designTokens.marginLG`（或等价 token），无硬编码数值。

### 2. 各步卡片正文

- **简短说明**：每卡一句 Text（type="secondary" 或 strong），描述该步作用或当前状态（如「上传招标文件」「解析完成，共 xxx 字」「请先完成解析」）。
- **主操作区**：仅在该步可操作时渲染主按钮（上传、开始解析、开始分析、开始参数提取、开始生成框架、接受并继续/重新生成/添加要点、开始按章生成、下载 Word）；不可操作时仅展示说明文案，不展示主按钮。
- **详情 Collapse**：解析预览、分析结果、参数（项目信息/BIM/风险点）、框架章节与要点、按章章节列表与每章操作，均放在对应卡片的 Collapse 内，且 **defaultActiveKey={[]}**（默认收起）。

### 3. 按钮与不可用说明

- 沿用现有 `showAnalyzeTrigger`、`showParamsTrigger`、`showFrameworkTrigger`、`showChaptersTrigger` 等逻辑；未满足条件时卡片内不显示主按钮，改为显示「请先完成 xxx」类文案。
- 失败步骤的「重试」按钮保留在对应卡片内（如解析失败、分析失败等）。

## 四、如何验证阶段 2.2

### 1. 验证「每步一卡片、顺序与头部」

1. 启动前端与后端（见 [开发前-服务启动顺序.md](开发前-服务启动顺序.md)）。
2. 进入任务详情页（如 `/tasks/1`），步骤条下方应看到 **7 张卡片**，顺序为：上传、解析、分析、参数、框架、按章生成、导出。
3. 每张卡片标题为步骤名，右上（或标题旁）为**状态 Tag**（待执行/进行中/待确认/已完成/失败），颜色与 stepStatus 约定一致。

### 2. 验证「简短说明与主操作」

- 未完成上传时：上传卡有「上传招标文件」说明与文件选择 + 上传按钮；解析卡为「请先完成上传」，无主按钮。
- 解析完成后：解析卡为「解析完成，共 xxx 字」；分析卡在可操作时为「可开始分析」+「开始分析」按钮，否则为「请先完成解析」等说明。
- 依此类推：参数、框架、按章生成、导出各卡在可操作时显示主按钮，不可操作时显示一句说明。

### 3. 验证「详情可折叠、默认收起」

- 解析完成：解析卡片内存在可展开的 Collapse（如「预览」），默认收起，点击可展开查看解析文本。
- 分析/参数/框架/按章完成：对应卡片内均有 Collapse（分析结果、项目信息/BIM/风险点、章节框架、各章预览），默认收起。

### 4. 验证「步骤条点击滚动仍有效」

- 点击步骤条某一步，页面应平滑滚动到对应**卡片**（因 Card 带 `id="step-{key}"`）。

### 5. 验收标准汇总

| 项 | 标准 |
|----|------|
| 按步骤分块 | 7 个步骤各对应一张 Card，卡片内有步骤名、状态 Tag、简短说明 |
| 主操作按钮 | 仅在该步可操作时展示；不可用时有一句明确说明（如请先完成上传/解析） |
| 详情可折叠 | 解析/分析/参数/框架/按章等长内容在卡片内 Collapse 中，默认收起 |
| 样式 | 卡片间距等使用 design token（如 marginLG），Tag 颜色来自 stepStatus |
