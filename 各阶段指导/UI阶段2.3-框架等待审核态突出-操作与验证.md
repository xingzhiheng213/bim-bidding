# UI 阶段 2.3 框架「等待审核」态突出 — 操作与验证指南

本文档面向执行与验收，按「阶段要达成什么 → 需要准备什么 → 如何验证」说明 **UI 设计计划** 阶段 2.3（框架「等待审核」态突出）的完整流程。阶段 2.3 在框架步骤为 `waiting_user` 时，在步骤条与对应卡片上做视觉突出，并在卡片内集中展示操作与提示文案，与设计系统语义色一致。

---

## 一、阶段 2.3 要达成什么

- **步骤条区域突出**：当框架步骤为 `waiting_user` 时，在步骤条与下方卡片列表之间渲染一条 **Alert（type="warning"）**，文案如「当前步骤：框架待确认 — 请确认框架或添加要点后继续。」，使用户打开详情页即可看到当前待确认步骤。
- **框架卡片视觉突出**：当框架步骤为 `waiting_user` 时，对应 **Card** 增加左侧强调边框（使用 `designTokens.colorWarning`）及可选浅色背景（如 `rgba(250, 173, 20, 0.06)`），与其它步骤卡片区分明显。
- **卡片内提示与操作集中**：在框架卡片内、操作按钮上方增加一条 **Alert type="warning"**，文案「请确认框架或添加要点后继续。」；操作区保持「接受并继续」「重新生成框架」「添加要点」「查看框架对比」集中展示，风格与设计系统一致。
- **交付物**：任务详情页在框架待确认时，步骤条下方有提示、框架卡片有边框/背景突出、卡片内有 Alert 与操作按钮；样式均使用 design token 或 Ant 语义色。

## 三、阶段 2.3 交付物说明

完成 2.3 后，应满足以下内容：

### 1. 步骤条下方 Alert（`TaskDetailPage.tsx`）

- **条件**：`frameworkStep?.status === 'waiting_user'` 时渲染。
- **组件**：`Alert type="warning"`，`showIcon`，文案「当前步骤：框架待确认 — 请确认框架或添加要点后继续。」
- **位置**：紧接在 `Steps` 组件之后、`TASK_STEP_ORDER.map` 卡片列表之前；间距使用 `designTokens.marginLG`。

### 2. 框架卡片样式（`TaskDetailPage.tsx`）

- **条件**：`stepKey === 'framework'` 且 `status === 'waiting_user'` 时，为该 Card 应用特殊样式。
- **样式**：`borderLeft: 4px solid ${designTokens.colorWarning}`；可选 `background: 'rgba(250, 173, 20, 0.06)'`。其它步骤卡片保持 `style={{ marginBottom: designTokens.marginLG }}`。
- **实现**：在 `TASK_STEP_ORDER.map` 内根据 `stepKey` 与 `status` 计算 `cardStyle`，再传给 Card 的 `style`。

### 3. 框架卡片内提示与操作区

- **提示**：在「接受并继续」「重新生成框架」「添加要点」等按钮**上方**增加 `Alert type="warning"`，文案「请确认框架或添加要点后继续。」，仅当 `frameworkStep?.status === 'waiting_user'` 且 `frameworkChapters.length > 0` 时渲染。
- **操作区**：保持现有三个主操作按钮 +「查看框架对比」的布局不变，与 Alert 同处一视觉块（同一 `div`），无需改交互逻辑。

## 四、如何验证阶段 2.3

### 1. 验证「步骤条下方提示」

1. 启动前端与后端（见 [开发前-服务启动顺序.md](开发前-服务启动顺序.md)）。
2. 进入一个**框架步骤为待确认**的任务详情页（即框架已生成完成、状态为 `waiting_user`、尚未点击「接受并继续」）。
3. 步骤条下方应出现一条**黄色/警告色 Alert**，文案为「当前步骤：框架待确认 — 请确认框架或添加要点后继续。」，带图标。

### 2. 验证「框架卡片视觉突出」

- 在同一任务详情页，找到「框架」步骤对应的卡片。
- 该卡片应有**左侧 4px 警告色边框**（与 design token colorWarning 一致），以及可选的浅色背景，与其它 6 张卡片明显区分。
- 当框架步骤为 `completed` 或其它状态时，框架卡片不应再带该边框/背景，仅保留常规 `marginBottom`。

### 3. 验证「卡片内 Alert 与操作」

- 框架步骤为 `waiting_user` 且已有章节列表时，框架卡片内、操作按钮上方应有一条 **Alert type="warning"**，文案「请确认框架或添加要点后继续。」。
- 其下为「接受并继续」「重新生成框架」「添加要点」「查看框架对比」按钮，布局与 2.2 一致，风格与设计系统一致。

### 4. 验收标准汇总

| 项 | 标准 |
|----|------|
| 步骤条区域 | 框架为 waiting_user 时，步骤条下方有一条 warning Alert，文案含「框架待确认」与「请确认框架或添加要点后继续」 |
| 框架卡片样式 | 框架为 waiting_user 时，该 Card 有左侧 4px 警告色边框及可选浅底；其它状态无此样式 |
| 卡片内提示 | 框架为 waiting_user 且有章节时，卡片内操作区上方有 warning Alert「请确认框架或添加要点后继续。」 |
| 设计 token | 边框色使用 designTokens.colorWarning，间距使用 designTokens.marginLG / marginSM |

## 五、与 2.1、2.2 的衔接

- **阶段 2.1**：2.3 不改变步骤条结构与点击滚动，仅在步骤条下方条件增加 Alert。
- **阶段 2.2**：2.3 在 2.2 的「每步一卡片」与框架卡片内操作区基础上，叠加步骤条下方 Alert、框架 Card 的边框/背景、以及卡片内 Alert 提示；操作按钮逻辑与布局保持不变。

若后续需要步骤条**节点本身**高亮（如当前 process 步骤为 framework 时给该节点加边框/背景），可在外层包 `data-step-current="framework"` 的 div 并配合 CSS 迭代实现；本阶段以步骤条下方 Alert + 卡片突出为主。
