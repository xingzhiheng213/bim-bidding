# UI 阶段 3.3 空态 — 操作与验证指南

本文档面向执行与验收，按「阶段要达成什么 → 需要准备什么 → 如何验证」说明 **UI 设计计划** 阶段 3.3（空态）的完整流程。阶段 3.3 增强首页无任务时的空态：使用图标/插画 + 引导文案 + 主按钮或明显入口，样式与 design token 一致。

---

## 一、阶段 3.3 要达成什么

- **无任务时**：不只有表格的 emptyText，增加插画或图标 + 一句引导文案（如「暂无任务，点击下方按钮创建第一个标书任务」）+ 主按钮或明显入口。
- **样式与设计系统一致**：颜色、间距、字体层级使用 design token 或 Ant 主题。
- **交付物**：首页空态友好；空态区域带图标/插画、引导文案、主按钮；样式符合设计系统。

## 三、阶段 3.3 交付物说明

完成 3.3 后，应满足以下内容：

### 1. 使用 Ant Design Empty 增强空态（`HomePage.tsx`）

- **引入**：从 `antd` 增加 `Empty` 的引入。
- **替换 emptyText**：将 Table 的 `locale.emptyText` 从字符串改为 **ReactNode**，内容为：
  - 外层 `div`：`style={{ padding: designTokens.paddingLG }}`，保证空态区域与列表区块内边距统一。
  - **Empty** 组件：`description` 为引导文案（如「暂无任务，点击下方按钮创建第一个标书任务」）；`extra` 为 **主按钮**「创建任务」，`onClick` 调用 `createMutation.mutate()`，`loading={createMutation.isPending}`，`type="primary"`。
  - **图标/插画**：Empty 自带默认插画，满足「图标或插画」要求；可选 `image={Empty.PRESENTED_IMAGE_SIMPLE}` 使用简版插画。

### 2. 样式与 design token 一致

- **间距**：空态区域使用 `designTokens.paddingLG`，不新增硬编码数值。
- **颜色与字体**：Empty 的 description 使用 Ant 默认次要文字色；主按钮为 `type="primary"`，符合设计系统。

## 四、如何验证阶段 3.3

### 1. 验证「空态展示」

1. 启动前端与后端（见 [开发前-服务启动顺序.md](开发前-服务启动顺序.md)）。
2. 打开首页，在**无任务**时（如新环境或已删除全部任务），任务列表区域应展示：
   - **插画/图标**：Empty 默认插画（或简版插画）。
   - **引导文案**：如「暂无任务，点击下方按钮创建第一个标书任务」。
   - **主按钮**：「创建任务」主按钮（蓝色）。

### 2. 验证「空态交互」

- 点击空态中的「创建任务」按钮，应创建任务并跳转到任务详情页，与首屏主操作「创建任务」行为一致。
- 加载中时按钮显示 loading 态。

### 3. 验证「样式与设计系统」

- 空态区域与表格区域视觉协调，内边距使用 design token（如 paddingLG），无硬编码色值/间距。

### 4. 验收标准汇总

| 项 | 标准 |
|----|------|
| 空态内容 | 无任务时展示插画/图标 + 引导文案 + 「创建任务」主按钮 |
| 空态交互 | 点击空态内主按钮可创建任务并跳转详情 |
| 设计 token | 空态区域间距使用 designTokens.paddingLG，样式与设计系统一致 |

## 五、与 3.1、3.2 的衔接

- **阶段 3.1**：3.3 不改变首页主标题、副标题、首屏「创建任务」与「任务列表」区块布局；空态仅在任务列表 Table 的 emptyText 内展示。
- **阶段 3.2**：3.3 不改变表格列定义与状态展示；空态仅在 dataSource 为空时由 Table 渲染，loading 时仍显示 Table 的 loading 态。
