# 阶段 0.1 后端骨架 — 操作与验证指南（新手向）

本文档面向新手，按「需要准备什么 → 一步步操作 → 如何验证」说明阶段 0.1 的完整流程。阶段 0.1 只做**后端骨架**（FastAPI + Celery + Redis + PostgreSQL），不做前端。

---

## 一、阶段 0.1 要达成什么

- 在电脑上能**启动后端服务**（FastAPI）和**异步任务 Worker**（Celery）。
- 访问一个**健康检查地址**有正常返回。
- 能**发一个演示任务**给 Celery，Worker 执行后脚本能拿到结果。
- **数据库**能连上（暂不建业务表）。

---

## 二、你需要提前准备的东西

| 项目 | 说明 | 如何确认 |
|------|------|----------|
| Python 3.10+ | 本机已安装 Python | 在终端输入 `python --version` 能看到版本号 |
| Redis | 用来给 Celery 发任务、存结果 | 见下文「三、1」 |
| PostgreSQL | 用来做业务数据库（后续阶段用） | 见下文「三、2」 |
| 项目代码 | 已有 `backend/` 目录及其中文件 | 打开 `标书工作流\backend` 能看到 `app`、`celery_app`、`tasks` 等文件夹 |

若你还没有 Redis / PostgreSQL，可按下面「三、1」「三、2」用 Docker 或本机安装。

---

## 三、环境准备（Redis 与 PostgreSQL）

### 1. Redis（Celery 任务队列）

**方式 A：用 Docker（推荐）**

在终端执行：

```bash
docker run -d --name redis-bim -p 6379:6379 redis:7-alpine
```

本机访问地址：`localhost:6379`，无密码。

**方式 B：本机安装**

- Windows 可安装 [Redis for Windows](https://github.com/microsoftarchive/redis/releases) 或通过 WSL 安装。
- 默认端口 6379，无密码。

**验证**：Redis 启动后，本机用 `redis://localhost:6379/0` 即可，后面在 `.env` 里会用到。

---

### 2. PostgreSQL（业务数据库）

**方式 A：用 Docker**

在终端执行（把 `你的密码` 换成你要设的密码，不要有特殊符号更好）：

```bash
docker run -d --name pg-bim -e POSTGRES_USER=postgres -e POSTGRES_PASSWORD=你的密码 -e POSTGRES_DB=bim_bidding -p 5432:5432 postgres:16-alpine
```

这样会**自动创建**数据库 `bim_bidding`，无需再执行 `CREATE DATABASE`。

**方式 B：本机安装 PostgreSQL**

1. 安装 [PostgreSQL](https://www.postgresql.org/download/windows/)。
2. 安装时记住给 **postgres** 用户设的密码。
3. 用 **pgAdmin** 或 **SQL Shell (psql)** 连上后，执行：
   ```sql
   CREATE DATABASE bim_bidding;
   ```
   （注意：`CREATE DATABASE` 是 SQL，要在 PostgreSQL 里执行，不能在 PowerShell 里直接输入。）

**验证**：能用 pgAdmin 连上本机 PostgreSQL，并看到数据库 `bim_bidding`。

## 四、后端操作步骤（按顺序做）

### 步骤 1：进入后端目录

在终端（PowerShell 或 CMD）中执行：

```powershell
cd d:\标书工作流\backend
```

（若你的项目不在 D 盘，改成你的实际路径。）

---

### 步骤 2：创建并激活虚拟环境

```powershell
python -m venv .venv
.venv\Scripts\activate
```

激活成功后，命令行前面会出现 `(.venv)`。

---

### 步骤 3：安装依赖

```powershell
pip install -r requirements.txt
```

等待安装完成，无报错即可。

---

### 步骤 4：配置环境变量（.env）

1. 在 `backend` 目录下找到 `.env.example`，复制一份并改名为 `.env`。
2. 打开 `.env`，按你的实际环境填写：

```env
# Redis（Celery 用）
REDIS_URL=redis://localhost:6379/0

# PostgreSQL（格式：postgresql://用户名:密码@主机:端口/数据库名）
DATABASE_URL=postgresql://postgres:你的postgres密码@localhost:5432/bim_bidding
```

说明：

- **REDIS_URL**：本机 Redis 无密码就用上面这行。
- **DATABASE_URL**：`postgres` 是用户名，`你的postgres密码` 换成你安装/创建容器时设的密码，`bim_bidding` 是数据库名。  
  例如密码是 `229999`，则写：  
  `DATABASE_URL=postgresql://postgres:229999@localhost:5432/bim_bidding`

保存 `.env` 后关闭。

---

### 步骤 5：启动 FastAPI（第一个终端）

在 **backend** 目录下、已激活 `.venv` 的情况下执行：

```powershell
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

看到类似 `Uvicorn running on http://0.0.0.0:8000` 和 `Application startup complete.` 即表示启动成功。

- 若出现 `Database connection check failed: no password supplied`：说明 **DATABASE_URL** 里没写密码或格式不对，检查 `.env` 是否为 `postgresql://postgres:密码@localhost:5432/bim_bidding`。
- 即使数据库连不上，服务也会启动，只是会打一条警告；把数据库配对后重启 uvicorn 即可。

---

### 步骤 6：启动 Celery Worker（第二个终端）

再开一个终端窗口：

```powershell
cd d:\标书工作流\backend
.venv\Scripts\activate
celery -A celery_app worker -l info -P solo
```

Windows 下建议加 `-P solo`。看到 `transport: redis://localhost:6379/0`、`results: redis://localhost:6379/0` 和 `celery@xxx ready.` 即表示 Worker 已连上 Redis 并就绪。

---

## 五、验证（阶段 0.1 验收）

按下面三项逐项检查，都通过即表示阶段 0.1 完成。

### 验证 1：健康检查接口

1. 保持 uvicorn 在运行（步骤 5 的终端不要关）。
2. 浏览器打开：<http://localhost:8000/health>
3. 页面上应显示：`{"status":"ok"}`

若打不开，检查：是否在 backend 下启动了 uvicorn、端口是否为 8000、防火墙是否拦截。

---

### 验证 2：数据库连接（可选但建议做）

1. 保持 uvicorn 在运行。
2. 看**启动 uvicorn 的那个终端**：若配置正确，会有一行 `Database connection OK`；若之前是 `Database connection check failed`，把 `.env` 里 `DATABASE_URL` 的密码改对后，**重启 uvicorn**，再看是否变成 OK。

---

### 验证 3：Celery 演示任务

1. 保持 **Celery Worker 在运行**（步骤 6 的终端不要关）。
2. 再开**第三个终端**，执行：

```powershell
cd d:\标书工作流\backend
.venv\Scripts\activate
python test_hello.py
```

3. 预期结果：
   - 脚本打印 Broker、Result backend 地址；
   - 打印 `Task sent, task_id: xxx`；
   - 几秒内打印 `Result: {'message': 'Hello, BIM!'}` 和 `OK`。
4. 同时看 **Worker 所在终端**，应出现：
   - `Task tasks.demo.hello [xxx] received`
   - `Task tasks.demo.hello [xxx] succeeded in 0.0xs: {'message': 'Hello, BIM!'}`

若脚本一直不出现 `Result:` 且最后报 Timeout：先确认 Worker 是否在运行、是否在 backend 下启动；若 Worker 有 received/succeeded 但脚本仍超时，说明已按「轮询取结果」改过 `test_hello.py`，一般再跑一次即可。

## 六、常见问题

| 现象 | 可能原因 | 处理办法 |
|------|----------|----------|
| `CREATE DATABASE` 在 PowerShell 里报错 | 那是 SQL，不是 Windows 命令 | 在 pgAdmin 或 psql 里执行 `CREATE DATABASE bim_bidding;` |
| Database connection check failed: no password supplied | DATABASE_URL 里没写密码或写错 | 写成 `postgresql://postgres:你的密码@localhost:5432/bim_bidding` |
| test_hello.py 一直不返回结果 / 超时 | Worker 没开，或客户端等结果方式在 Windows 上有问题 | 先保证 Worker 在跑；脚本已改为轮询取结果，一般可解决 |
| pgAdmin 里看不到数据库 | 还没添加/连接服务器 | 在 pgAdmin 里点「Add New Server」，填 Host、Port、用户名、密码，连上后再在「Databases」下看到库并创建 `bim_bidding` |

## 七、小结：阶段 0.1 检查清单

- [ ] Redis 已启动，`REDIS_URL=redis://localhost:6379/0` 可用。
- [ ] PostgreSQL 已安装/启动，已创建数据库 `bim_bidding`，`.env` 里 `DATABASE_URL` 格式正确且密码正确。
- [ ] 在 `backend` 下创建并激活了 `.venv`，已执行 `pip install -r requirements.txt`。
- [ ] 复制并填写了 `.env`（REDIS_URL、DATABASE_URL）。
- [ ] 在 backend 下执行 `uvicorn app.main:app --reload --host 0.0.0.0 --port 8000` 能启动，访问 http://localhost:8000/health 得到 `{"status":"ok"}`。
- [ ] 在 backend 下执行 `celery -A celery_app worker -l info -P solo` 能启动，看到 `ready` 和 `redis://localhost:6379/0`。
- [ ] 在 Worker 运行的情况下执行 `python test_hello.py`，能打印出 `Result: {'message': 'Hello, BIM!'}` 和 `OK`，且 Worker 终端有 received/succeeded。

以上全部打勾后，阶段 0.1 后端骨架即算完成，可进入阶段 0.2（前端骨架）或阶段 0.3（基础数据模型）。
