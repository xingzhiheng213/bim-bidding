# 阶段 1.1 上传接口与存储 — 操作与验证指南（新手向）

本文档面向新手，按「需要准备什么 → 一步步操作 → 如何验证」说明阶段 1.1 的完整流程。阶段 1.1 只做**后端上传接口与存储**：`POST /api/tasks/{task_id}/upload` 接收文件，校验类型与大小，落盘保存，并更新该任务的 upload 步骤（不包含前端上传界面与文档解析，前端在 1.3，解析在 1.2）。

---

## 一、阶段 1.1 要达成什么

- 后端提供 **POST /api/tasks/{task_id}/upload**，接收 multipart 文件，校验类型（pdf/doc/docx）与大小（默认最大 50MB）。
- 将文件保存到**本地目录**（如 `backend/data/uploads/task_{id}/`），路径由配置 `UPLOAD_DIR` 决定。
- 创建或更新该任务的 **upload** 步骤：状态为 completed，在 **output_snapshot** 中记录相对路径与原始文件名（不暴露本机绝对路径）。
- 验收时能用 **curl 或 Postman** 完成：创建任务 → 上传文件 → 查看任务详情与磁盘文件。

**验收标准**（来自开发计划）：能对已有任务调用上传接口并返回成功；该任务的 upload 步骤状态为 completed，output_snapshot 中存有文件信息；GET /api/tasks/{id} 可看到上述信息；磁盘上在 UPLOAD_DIR/task_{id}/ 下存在对应文件。

## 三、环境准备

- **后端依赖**：阶段 1.1 增加了 `python-multipart`（FastAPI 文件上传需要）。在 `backend` 目录下、激活虚拟环境后执行：  
  `pip install -r requirements.txt`  
  或至少：`pip install python-multipart`
- **环境变量（可选）**：若需自定义上传目录或大小限制，在 `backend/.env` 中增加（与 `.env.example` 一致即可）：  
  `UPLOAD_DIR=data/uploads`  
  `MAX_UPLOAD_SIZE_MB=50`  
  不配置则使用代码中的默认值（同上）。

## 四、操作步骤（按顺序做）

### 步骤 1：启动后端（第一个终端）

在终端（PowerShell 或 CMD）中执行：

```powershell
cd d:\标书工作流\backend
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8001
```

看到 `Uvicorn running on http://0.0.0.0:8001` 和 `Application startup complete.` 即表示后端就绪。**此窗口不要关**。

---

### 步骤 2：创建一条任务（第二个终端，用 curl）

再开一个终端，执行（创建任务）：

```powershell
curl -X POST http://localhost:8001/api/tasks -H "Content-Type: application/json" -d "{}"
```

记下返回 JSON 里的 **id**（例如 `1`）。后面把「任务 id」都换成这个数字。

若没有 curl，可用 PowerShell：

```powershell
Invoke-RestMethod -Method Post -Uri "http://localhost:8001/api/tasks" -ContentType "application/json" -Body "{}"
```

同样记下返回的 `id`。

---

### 步骤 3：上传文件到该任务

将下面命令里的 **任务 id** 和 **文件路径** 换成你的实际值后执行。

```powershell
curl -X POST http://localhost:8001/api/tasks/1/upload -F "file=@C:\Users\你的用户名\Desktop\你的文件.pdf"
```

- 把 **`/1/`** 换成步骤 2 得到的 id（例如 id 为 2 则写 `/2/`）。
- 把 **`@C:\Users\你的用户名\Desktop\你的文件.pdf`** 换成你电脑上 PDF 或 DOCX 的**完整路径**；路径含空格时用英文双引号包住整体。

成功时返回 **201**，body 中应有 `"step_key": "upload"`、`"status": "completed"`、`"stored_path": "task_1/xxx.pdf"` 等。

---

### 步骤 4：查看任务详情（确认 upload 步骤）

```powershell
curl http://localhost:8001/api/tasks/1
```

（把 `1` 换成你的任务 id。）

在返回的 JSON 里找到 **steps** 中 **step_key 为 "upload"** 的那条，应满足：

- **status** 为 **"completed"**。
- **output_snapshot** 为一段 JSON 字符串，内含 **stored_path**（相对路径，如 `task_1/xxx.pdf`）、**original_filename**（你传的文件名），且**不包含**本机绝对路径（如 `D:\...`）。

---

### 步骤 5：确认文件已落盘

打开文件资源管理器，进入：

`d:\标书工作流\backend\data\uploads\task_1\`

（若任务 id 为 2，则为 `task_2`。）

该目录下应有一个文件（文件名为一串英文数字 + .pdf 或 .docx），即刚上传的文件。若没有 `data\uploads` 或 `task_1`，检查是否在 `backend` 目录下运行 uvicorn，或查看 `.env` 中是否配置了 `UPLOAD_DIR`（默认相对 backend 的 `data/uploads`）。

## 五、验证（阶段 1.1 验收）

按下面三项逐项检查，都通过即表示阶段 1.1 完成。

### 验证 1：上传接口返回成功且步骤更新

1. 对已存在的任务 id 调用 `POST /api/tasks/{task_id}/upload`，上传合法 PDF 或 DOCX（且小于 50MB）。
2. 应返回 **201**，body 含 `step_key`、`status`、`stored_path`（相对路径）。
3. 任务不存在时返回 **404**；文件类型错误或超大时返回 **400**。

---

### 验证 2：任务详情中 upload 步骤正确

1. 调用 `GET /api/tasks/{id}`（id 为上一步使用的任务 id）。
2. 在 **steps** 中找到 **step_key 为 "upload"** 的项。
3. 其 **status** 为 **"completed"**，**output_snapshot** 为合法 JSON，包含 **stored_path**、**original_filename**，且**不包含**本机绝对路径。

---

### 验证 3：磁盘上存在上传文件

1. 打开 `backend/data/uploads/`（或你在 `.env` 中配置的 `UPLOAD_DIR`）。
2. 进入 **task_{任务id}** 目录。
3. 其中应有一个刚上传的文件（名称可能为 uuid+后缀）。

## 六、常见问题

| 现象 | 可能原因 | 处理办法 |
|------|----------|----------|
| curl 返回 404 | 任务 id 写错或任务不存在 | 先用 POST /api/tasks 创建任务，再用返回的 id 做 upload |
| curl 返回 400 | 文件类型或大小不符 | 仅支持 .pdf/.doc/.docx；大小不超过 MAX_UPLOAD_SIZE_MB（默认 50MB） |
| 找不到 data\uploads | 未上传过或 UPLOAD_DIR 配置不同 | 上传成功后再看；或查看 config 中 UPLOAD_DIR 的实际路径 |
| 后端报错 ModuleNotFoundError: multipart | 未安装 python-multipart | 在 backend 下执行 `pip install -r requirements.txt` 或 `pip install python-multipart` |
| output_snapshot 里是绝对路径 | 与设计不符 | 当前实现应只存相对路径；若有绝对路径请检查代码 |
