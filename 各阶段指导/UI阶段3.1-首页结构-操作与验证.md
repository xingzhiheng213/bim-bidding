# UI 阶段 3.1 首页结构 — 操作与验证指南

本文档面向执行与验收，按「阶段要达成什么 → 需要准备什么 → 如何验证」说明 **UI 设计计划** 阶段 3.1（首页结构）的完整流程。阶段 3.1 调整首页结构：主操作（创建任务）在首屏突出、标题与可选副标题清晰、任务列表区块与主操作有明确间距与层级，并统一使用 design token。

---

## 一、阶段 3.1 要达成什么

- **主操作突出**：**创建任务** 按钮在首屏显著位置，使用设计系统主按钮（`type="primary"`），与标题区、任务列表区块的间距使用 design token，分区明确。
- **标题与说明**：首页保留一个主标题（如「我的任务」），主标题下方可选副标题（如「上传招标文件，一键生成标书」），使用 `Typography.Text type="secondary"`，不重复堆砌。
- **任务列表区块**：区块标题为「任务列表」，与上方主操作有明确间距与层级（如 `marginTop: marginXL`）；标题与表格之间使用 token（如 `marginSM`）。
- **交付物**：首页结构清晰；主操作与列表分区明确；全页间距使用 design token，无硬编码数值。

## 三、阶段 3.1 交付物说明

完成 3.1 后，应满足以下内容：

### 1. 引入 design token（`HomePage.tsx`）

- 顶部增加：`import { designTokens } from '../theme/tokens'`。
- 所有区块间距、标题边距使用 token（如 `marginLG`、`marginXL`、`marginSM`），不再使用硬编码 16/24/12。

### 2. 主标题与副标题

- **主标题**：保留一个，如「我的任务」（`Title level={2}`），下边距 `designTokens.marginLG`。
- **副标题**：主标题下方一行 `Typography.Text type="secondary"`，文案「上传招标文件，一键生成标书」；与主操作（创建任务按钮）的间距使用 `designTokens.marginLG`。

### 3. 主操作（创建任务）

- 按钮保持 `type="primary"`，置于标题（+ 副标题）之下、任务列表区块之上。
- 与下方「任务列表」区块的间距使用 `designTokens.marginLG`（或 `marginXL`），使主操作与列表分区明确。

### 4. 任务列表区块

- **区块标题**：「任务列表」（`Title level={5}`），与上方「创建任务」的间距使用 `designTokens.marginXL`（或 `marginLG`）。
- **标题与表格**：列表标题与 Table 之间使用 `designTokens.marginSM`。

## 四、如何验证阶段 3.1

### 1. 验证「首屏结构与主操作」

1. 启动前端与后端（见 [开发前-服务启动顺序.md](开发前-服务启动顺序.md)）。
2. 打开首页（如 `/`），首屏应依次看到：主标题「我的任务」、副标题「上传招标文件，一键生成标书」、**创建任务** 主按钮（蓝色主按钮）、「任务列表」标题、表格。
3. 「创建任务」按钮在视觉上突出，与标题区、列表区有清晰分区。

### 2. 验证「间距与 token」

- 主标题与副标题、副标题与按钮、按钮与「任务列表」、列表标题与表格之间的间距应统一、层次分明，无硬编码 16/24/12（可通过代码检查确认使用 `designTokens.*`）。

### 3. 验收标准汇总

| 项 | 标准 |
|----|------|
| 主操作突出 | 创建任务为主按钮，位于标题区之下、任务列表之上，分区明确 |
| 标题与副标题 | 主标题一个，副标题可选且为 secondary 样式，不堆砌 |
| 任务列表区块 | 区块标题「任务列表」，与主操作有明确间距与层级 |
| 设计 token | 全页间距使用 designTokens（marginLG、marginXL、marginSM 等） |
