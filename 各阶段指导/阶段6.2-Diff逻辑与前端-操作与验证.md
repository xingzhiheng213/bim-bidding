# 阶段 6.2 Diff 逻辑与前端 — 操作与验证指南（新手向）

本文档面向新手，按「需要准备什么 → 一步步操作 → 如何验证」说明阶段 6.2 的完整流程。阶段 6.2 为**审核专家模块的 Diff 逻辑与前端**：后端保持 6.1 行级 diff；前端新增「根据 diff 结果渲染：删除标红、新增标绿」的组件与对比页，满足阶段 6 验收「输入两段文本能生成 diff，并在前端正确显示标红标绿」。

---

## 一、阶段 6.2 要达成什么

- **后端**：保持 6.1 的 difflib 行级 diff，不修改；可选后续引入 diff-match-patch 或块级/LLM 语义，6.2 不实现。
- **前端 diff 渲染**：根据 6.1 返回的 `diff: Array<{ type: 'add'|'del', text: string }>` 在页面上渲染：**删除标红**（背景红 + 删除线）、**新增标绿**（背景绿）；换行正确（`whiteSpace: pre-wrap`）。
- **前端 API 封装**：`postCompare(original, modified)`、`getFrameworkDiff(taskId)`、`getChaptersDiff(taskId, chapterNumber)`，供对比页与后续 6.3 审核页调用。
- **验收入口**：单独「对比」页（路由 `/compare`），两个文本框（修改前/修改后）+「对比」按钮，调用 `postCompare` 后在下方用 `DiffView` 展示标红标绿。
- **验收标准**（来自开发计划）：输入两段文本能生成 diff，并在前端正确显示标红标绿。

## 三、环境准备

- **后端**：按 [开发前-服务启动顺序.md](开发前-服务启动顺序.md) 启动后端（uvicorn）；测对比页只需后端 + 前端，无需 Celery。
- **前端**：在 `frontend` 目录执行 `npm run dev`，浏览器可访问 http://localhost:5173。

---

## 四、操作步骤（按顺序做）

### 步骤 1：启动后端与前端

1. 终端 1：进入 `backend`，激活虚拟环境，执行 `uvicorn app.main:app --reload --host 0.0.0.0 --port 8000`。
2. 终端 2：进入 `frontend`，执行 `npm run dev`。
3. 浏览器打开 http://localhost:5173，应看到「BIM 标书生成」及首页。

---

### 步骤 2：进入「对比」页

1. 在首页或任意页顶部导航中，点击 **「对比」** 链接。
2. 应进入 http://localhost:5173/compare，页面标题为「文本对比（标红删除、标绿新增）」，有两个文本框（修改前、修改后）和一个「对比」按钮。

---

### 步骤 3：验证标红标绿

1. 在「修改前（original）」框中输入（可复制）：
   ```
   hello
   world
   ```
2. 在「修改后（modified）」框中输入：
   ```
   hello
   new
   world
   ```
3. 点击 **「对比」** 按钮。
4. 下方应出现「对比结果」区域：**红色背景 + 删除线** 表示删除（本例中若行级 diff 将「原第二行」标为删除、「new」标为新增，则会出现红色块与绿色块）；**绿色背景** 表示新增（`new` 一行）。
5. 若两段完全一致，点击「对比」后可能显示「两段文本无差异」或对比结果为空；可故意改几个字再测，确认红/绿块出现且换行正确。

## 五、验证（阶段 6.2 验收）

按下面两项逐项检查，都通过即表示阶段 6.2 完成。

### 验证 1：对比页能调通 POST /api/compare 并展示 diff

1. 打开「对比」页，输入修改前、修改后两段文本，点击「对比」。
2. 无报错（无「对比请求失败」或 Network Error）；下方出现「对比结果」区域。
3. 当两段有差异时，结果区域中能看到**红色块**（删除）和**绿色块**（新增），且换行与输入一致。

### 验证 2：标红标绿与阶段 6 验收一致

1. 删除内容：仅在「修改前」存在的文本，应在结果中以**红色背景 + 删除线**显示。
2. 新增内容：仅在「修改后」存在的文本，应在结果中以**绿色背景**显示。
3. 阶段 6 验收标准「输入两段文本能生成 diff，并在前端正确显示标红标绿」即满足。

## 六、常见问题

| 现象 | 可能原因 | 处理办法 |
|------|----------|----------|
| 点击「对比」后报「对比请求失败」或 Network Error | 后端未启动或前端请求地址错误 | 确认后端已启动（http://localhost:8000/health 正常）；检查前端 `VITE_API_BASE`（如 `.env.development`）是否指向后端地址 |
| 对比结果区域空白或一直无红绿块 | 两段文本完全一致，或 diff 为空 | 故意在修改后增删一两行再点「对比」；或检查后端 POST /api/compare 是否返回了 `diff` 数组 |
| 顶部没有「对比」链接 | 前端未更新或未部署 6.2 | 确认 `frontend/src/App.tsx` 中有 `/compare` 路由，且 `HomePage`/`TaskDetailPage`/`SettingsPage` 等头部有「对比」链接 |
| 红/绿块换行错乱 | 样式未保留换行 | 确认 `DiffView` 使用 `whiteSpace: 'pre-wrap'`；若仍错乱可检查是否被父容器样式覆盖 |
