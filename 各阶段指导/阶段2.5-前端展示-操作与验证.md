# 阶段 2.5 前端展示 — 操作与验证指南（新手向）

本文档面向新手，按「需要准备什么 → 一步步操作 → 如何验证」说明阶段 2.5 的完整流程。阶段 2.5 聚焦**任务详情页的前端展示**：在任务详情中按步骤展示「解析 → 分析 → 参数 → 框架」，每步完成后展示对应结果（分析结果可折叠长文本，框架展示为章节列表）；与阶段 2.4 验收衔接，确保上传并解析后能一键跑完「分析 → 参数 → 框架」并在前端看到章节框架列表。

---

## 一、阶段 2.5 要达成什么

- **按步骤展示**：在任务详情页中，步骤顺序清晰可见：**上传与解析** → **分析** → **参数提取** → **框架生成**；各步骤的触发按钮、进行中提示、完成结果与失败重试均在对应区块展示。
- **解析完成**：解析（extract）完成后展示「解析完成，共 xxx 字」及可展开的**预览**（Collapse）。
- **分析完成**：分析（analyze）完成后展示「分析结果」为**可折叠长文本**（Collapse，内容可展开）。
- **参数完成**：参数提取（params）完成后展示**项目信息**、**BIM 要求**、**风险点**（可折叠或分块）。
- **框架完成**：框架（framework）完成后展示**章节列表**（第 N 章 标题），可折叠展示。
- **验收衔接**：与阶段 2 整体验收一致：上传并解析后，能一键跑完「分析 → 参数 → 框架」，前端能看到章节框架列表；结构满足后续「按章生成」需要。

**验收标准**（来自开发计划）：在任务详情中按步骤展示解析→分析→参数→框架；分析结果可折叠长文本；框架展示为章节列表。

## 三、环境准备

- **无新增依赖**：前端沿用现有依赖（React、Ant Design、TanStack Query 等），无需额外 `npm install`。
- **前端 API 地址**：`frontend/.env.development` 或 `.env.local` 中 `VITE_API_BASE=http://localhost:8000`（与后端一致）。
- **服务顺序**：需要 **3 个终端**：后端（uvicorn）、Celery Worker、前端（npm run dev）。详见 [开发前-服务启动顺序.md](开发前-服务启动顺序.md)。

## 四、操作步骤（按顺序做）

### 步骤 1：确认 Redis、PostgreSQL 已运行

- **Redis**：CMD 或 PowerShell 执行 `redis-cli ping`，看到 `PONG`。
- **PostgreSQL**：pgAdmin 能连上本机或服务里 PostgreSQL 为「正在运行」。

若未运行，按 [开发前-服务启动顺序.md](开发前-服务启动顺序.md) 的「二」先启动。

---

### 步骤 2：启动后端（终端 1）

在终端（PowerShell 或 CMD）中执行：

```powershell
cd d:\标书工作流\backend
.venv\Scripts\activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

看到 `Uvicorn running on http://0.0.0.0:8000` 和 `Application startup complete.` 即表示后端就绪。**此窗口不要关**。

---

### 步骤 3：启动 Celery Worker（终端 2）

**再开一个** CMD 或 PowerShell，执行：

```powershell
cd d:\标书工作流\backend
.venv\Scripts\activate
celery -A celery_app worker -l info -P solo
```

看到 `celery@xxx ready.` 即表示 Worker 就绪。**此窗口不要关**。

---

### 步骤 4：启动前端（终端 3）

**再开一个** CMD 或 PowerShell，执行：

```powershell
cd d:\标书工作流\frontend
npm run dev
```

看到 `Local: http://localhost:5173/` 即表示前端就绪。**此窗口不要关**。

---

### 步骤 5：在任务详情页按顺序执行并确认展示

1. 浏览器打开 <http://localhost:5173>，点击**「创建任务」**，进入任务详情页（URL 为 `/tasks/数字`）。
2. **上传与解析**：
   - 在「上传招标文件」区块选择本机一个 **PDF 或 DOCX**，点击**「上传」**，等待上传成功（会自动触发解析）。
   - 等待「解析完成，共 xxx 字」出现，并确认有**可展开预览**（点击「预览」折叠面板可展开/收起）。
3. **分析**：点击**「开始分析」**，等待「分析中，请稍候…」变为「分析完成。」；确认出现**可折叠的「分析结果」**（Collapse，点击可展开长文本）。
4. **参数提取**：点击**「开始参数提取」**，等待「参数提取中，请稍候…」变为「参数提取完成。」；确认出现**项目信息**、**BIM 要求**、**风险点**（可折叠或分块展示）。
5. **框架生成**：点击**「开始生成框架」**，等待「框架生成中，请稍候…」变为「框架生成完成。」；确认出现**可折叠的「章节框架」**，展开后为**章节列表**（第 1 章 xxx、第 2 章 xxx…）。
6. **步骤顺序**：自顶向下确认页面区块顺序为：上传 → 解析结果 → 分析（按钮/结果）→ 参数提取（按钮/结果）→ 框架生成（按钮/结果）→ 步骤列表表格。

若某步失败，页面应展示错误信息及**「重试」**按钮；轮询在 extract/analyze/params/framework 为 running 时约每 2 秒刷新，无需手动刷新页面。

---

### 步骤 6：（可选）仅验证已有任务

若已有完成到「框架生成」的任务，可直接打开该任务详情页（如 http://localhost:5173/tasks/1），逐项确认：解析完成展示字数与预览、分析结果可折叠、参数三项分块、框架章节列表；无需重新上传与跑全流程。

---

## 五、验证（阶段 2.5 验收）

按下面五项逐项检查，都通过即表示阶段 2.5 完成。

### 验证 1：任务详情页可见顺序步骤（上传/解析 → 分析 → 参数 → 框架）

1. 在任务详情页，自顶向下查看主内容区。
2. 区块顺序应为：**上传招标文件**（或已上传提示）→ **解析结果**（解析完成/解析中/解析失败）→ **分析**（开始分析按钮/分析结果/分析失败）→ **参数提取**（开始参数提取按钮/参数结果/失败）→ **框架生成**（开始生成框架按钮/章节框架/失败）→ **步骤列表**表格。
3. 文案或区块顺序与「解析 → 分析 → 参数 → 框架」一致，且无顺序颠倒或缺失。

---

### 验证 2：解析完成后展示「解析完成，共 xxx 字」及可展开预览

1. 在解析（extract）步骤完成后，页面应显示「解析完成，共 xxx 字」（xxx 为解析出的文本字数）。
2. 下方应有**可展开的预览**（如 Collapse 组件，标题为「预览」），点击可展开/收起，内容为解析文本的前若干字。

---

### 验证 3：分析完成后展示「分析结果」且为可折叠长文本

1. 在分析（analyze）步骤完成后，页面应显示「分析完成。」。
2. 下方应有**可折叠的「分析结果」**（Collapse），点击可展开/收起；内容为分析产出的长文本，支持长文本折叠（如 ellipsis 可展开）。

---

### 验证 4：参数完成后展示项目信息、BIM 要求、风险点（可折叠或分块）

1. 在参数提取（params）步骤完成后，页面应显示「参数提取完成。」。
2. 下方应展示**项目信息**、**BIM 要求**、**风险点**三块内容（可为多个 Collapse 面板或分块展示）；内容与 params 步骤的 output_snapshot 一致（可为空列表或「（无）」）。

---

### 验证 5：框架完成后展示章节列表（第 N 章 标题）

1. 在框架（framework）步骤完成后，页面应显示「框架生成完成。」。
2. 下方应有**可折叠的「章节框架」**，展开后为**章节列表**，每项格式为「第 N 章 标题」（与 framework 步骤 output_snapshot.chapters 一致）。
3. 若 chapters 为空，可显示「框架已生成，但未解析到章节」等提示；步骤列表中 framework 的 status 为 completed。

## 六、常见问题

| 现象 | 可能原因 | 处理办法 |
|------|----------|----------|
| 页面显示「后端请求失败」或任务加载失败 | 后端未启动或端口不对 | 先在 backend 下启动 uvicorn（端口 8000）；检查 `frontend` 的 `VITE_API_BASE` 是否为 `http://localhost:8000` |
| 解析/分析/参数/框架一直不更新 | 未开 Celery Worker 或轮询未触发 | 开终端在 backend 下执行 `celery -A celery_app worker -l info -P solo`；确认任务详情页在步骤为 running 时约每 2 秒自动刷新 |
| 某步骤结果区块未出现 | 该步骤未完成或 output_snapshot 格式异常 | 用 GET /api/tasks/{id} 查看对应 step 的 status 与 output_snapshot；前端仅在有 output 且解析成功时展示结果区块 |
| 分析结果或章节框架展开后内容不全 | 前端对长文本做了截断或折叠 | 属正常；验证时确认「可折叠」「可展开」行为存在即可，无需强制展示全文 |
| 步骤顺序与文档不一致 | 前端布局或条件渲染顺序被改过 | 对照本文档「验证 1」的区块顺序检查 `TaskDetailPage.tsx` 中各组件的渲染顺序 |
