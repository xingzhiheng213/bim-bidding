# 阶段 5.2 进度与错误 — 操作与验证指南（新手向）

本文档面向新手，按「要达成什么 → 操作步骤 → 如何验证」说明阶段 5.2 的完整流程。阶段 5.2 为**进度与错误**：步骤失败时在 `task_steps` 中记录 `error_message`；前端展示错误信息及「重试」按钮（重新入队该步骤）；长任务显示进度条或「第 X 章 / 共 Y 章」。

---

## 一、阶段 5.2 要达成什么

- **后端**：步骤失败时在 `task_steps` 中记录 `error_message`（已由各 Celery 任务实现）；可选在任务上维护 `progress`（0–100），当前进度由 chapters 步骤的 `output_snapshot` 提供。
- **前端**：步骤失败时展示错误信息及「重试」按钮，点击后重新入队该步骤；长任务（按章生成）显示进度条或「第 X 章 / 共 Y 章」（阶段 4.4 已实现）。
- **验收标准**（来自开发计划）：步骤失败可重试且界面有反馈；长任务有进度展示。

## 三、操作步骤（按顺序做）

### 步骤 1：确认服务已启动

按 [开发前-服务启动顺序.md](开发前-服务启动顺序.md) 依次启动：后端（uvicorn）、Celery Worker、前端（npm run dev）。浏览器可访问 http://localhost:5173。

---

### 步骤 2：验证步骤失败时展示错误信息与重试

1. 进入一个任务详情页，使某一步骤失败（例如：在设置中不配置智谱 API Key，将「分析」步骤选为智谱模型，触发分析，分析步骤会失败）。
2. 确认该步骤区块内出现**错误 Alert**，展示「xxx 失败」及 `error_message` 内容。
3. 确认 Alert 右侧或下方有**「重试」按钮**；点击后应提示「已入队」或类似，轮询后步骤变为 running 再 completed 或再次 failed。
4. **解析步骤**：若解析失败（如文件格式异常或解析报错），应同样看到「解析失败」Alert 及**「重试」按钮**，点击后可重新入队解析。

---

### 步骤 3：验证长任务进度展示

1. 进入一个框架已确认的任务，选择全部或部分章节，点击「开始按章生成」。
2. 确认出现「正在生成第 X 章 / 共 Y 章…」及**进度条**，进度随已完成章数增加（阶段 4.4 已实现）。

---

## 四、验证（阶段 5.2 验收）

按下面三项逐项检查，都通过即表示阶段 5.2 完成。

### 验证 1：步骤失败时在 task_steps 中有 error_message

- 任一步骤（解析、分析、参数、框架、按章）失败后，通过 API `GET /api/tasks/{id}` 或数据库查看对应 `task_steps` 行，`error_message` 应有内容（且不为空）。

### 验证 2：步骤失败时前端展示错误信息及「重试」按钮

- 解析、分析、参数、框架、按章五步在失败时均显示错误 Alert 与「重试」按钮；点击重试后该步骤重新入队（状态先变为 running，再 completed 或再次 failed）。

### 验证 3：长任务显示进度条或「第 X 章 / 共 Y 章」

- 按章生成进行中时，页面展示进度条与「正在生成第 X 章 / 共 Y 章…」（或等价文案）；全部完成后展示「生成完成。」与「下载 Word」按钮（阶段 4.4）。

## 五、常见问题

| 现象 | 可能原因 | 处理办法 |
|------|----------|----------|
| 解析失败后没有「重试」按钮 | 前端未补全解析步骤的 Alert action | 确认 [TaskDetailPage.tsx](frontend/src/pages/TaskDetailPage.tsx) 中解析失败 Alert 带 `action={<Button ...>重试</Button>}`，且 `extractRunMutation` 已定义 |
| 点击重试无反应或报错 | 未满足该步骤的前置条件（如分析需解析完成） | 确认前置步骤为 completed；若为 4xx，查看接口返回的 detail |
| 进度条不更新 | 轮询未拿到最新 chapters 步骤 output_snapshot | 确认后端按章任务在每章完成后写入 `output_snapshot` 的 chapters/total/current；稍等几秒再观察 |
