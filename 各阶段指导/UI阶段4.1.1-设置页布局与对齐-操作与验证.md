# UI 阶段 4.1.1 设置页布局与对齐 — 操作与验证指南

本文档面向执行与验收，按「阶段要达成什么 → 需要准备什么 → 如何验证」说明 **UI 设计计划** 阶段 4.1.1（设置页布局与对齐）的完整流程。阶段 4.1.1 在 4.1 分区与样式不变的前提下，仅做布局调整：统一内容区宽度与框体对齐、表单项标签与控件对齐，不改变功能与分区。

---

## 一、阶段 4.1.1 要达成什么

- **统一内容区宽度**：设置页最外层包一层容器（如 maxWidth: 640，width: '100%'），主标题「设置」与两张 Card 均在该宽度内，左右边界一致，框体对齐。
- **大模型 API 区**：同一 provider 内 API Key 与 Base URL 输入框宽度统一（如都 400），框体对齐。
- **模型配置区**：默认模型与各步骤的「标签 + Select」采用固定标签宽度（如 100px），使所有 Select 左边缘对齐，标签列对齐。
- **框体统一**：两张 Card 宽度一致（width: '100%' 由外层容器约束），与标题同「框」，无单独 maxWidth 导致视觉错位。
- **交付物**：设置页框体与表单项对齐；内容区宽度统一；布局调整不改变 4.1 的分区与功能。

## 三、阶段 4.1.1 交付物说明

完成 4.1.1 后，应满足以下内容：

### 1. 统一内容区宽度与框体（`SettingsPage.tsx`）

- **外层容器**：主标题「设置」与两张 Card 包在一个容器 div 内，容器样式 `style={{ maxWidth: 640, width: '100%' }}`（或 720），使标题与 Card 左右边界一致。
- **Card 宽度**：两张 Card 的 style 去掉 `maxWidth: 560`，改为 `width: '100%'`，由外层容器约束宽度，与标题同「框」。

### 2. 大模型 API 区：输入框宽度统一

- **API Key 与 Base URL**：同一 provider 内 API Key 输入框与 Base URL 输入框宽度一致，如均为 `width: 400`（或 `width: '100%', maxWidth: 400`），保证两行输入框左对齐且右边界一致。
- **保存按钮**：位置与逻辑不变。

### 3. 模型配置区：标签与 Select 对齐

- **每行布局**：默认模型行与各步骤行采用「固定标签宽 + 控件」两列：每行外层 `display: flex`，左侧标签容器固定宽度（如 `width: 100`、`minWidth: 100`、`flexShrink: 0`），右侧为 Select（及可选「当前：xxx」）。
- **效果**：所有 Select 左边缘在同一竖线，标签列对齐。

### 4. 框体统一

- 两张 Card 宽度一致（均为 `width: '100%'`），与标题在同一内容宽度内，无单独 maxWidth 导致视觉错位。

## 四、如何验证阶段 4.1.1

### 1. 验证「统一内容区宽度」

1. 启动前端与后端（见 [开发前-服务启动顺序.md](开发前-服务启动顺序.md)）。
2. 打开设置页（如 `/settings`），在宽屏下（如 > 800px）观察：主标题「设置」与两张 Card 应有统一的左右边界（如最大宽度 640px 居中或左对齐），框体对齐，右侧无错位。

### 2. 验证「API 区输入框宽度」

- 大模型 API Card 内，每个 provider 的「API Key」与「Base URL」两行输入框宽度一致，左对齐且右边界一致。

### 3. 验证「模型区 Select 对齐」

- 模型配置 Card 内，「默认模型」与「分析」「参数提取」「框架生成」「按章生成」各行的 Select 左边缘在同一竖线；标签列（默认模型、分析等）左对齐。

### 4. 验收标准汇总

| 项 | 标准 |
|----|------|
| 内容区宽度 | 设置页有统一内容宽度（如 640px），标题与两张 Card 左右对齐 |
| API 输入框 | 同一 provider 内 API Key 与 Base URL 输入框宽度一致 |
| 模型区对齐 | 所有 Select 左边缘对齐，标签列固定宽 |
| 框体统一 | 两张 Card 宽度一致、与标题同「框」 |
