# 阶段 4.3 DOCX 导出 — 操作与验证指南（新手向）

本文档面向新手，按「需要准备什么 → 一步步操作 → 如何验证」说明阶段 4.3 的完整流程。阶段 4.3 实现**DOCX 导出**：将阶段 4.2 聚合出的 Markdown（项目信息 + 各章内容 + 风险点附录）转换为 Word 文档（.docx），并提供**下载接口**；前端在「按章生成完成」后展示**「下载 Word」**按钮，点击即可下载完整标书文档。

---

## 一、阶段 4.3 要达成什么

### 目标

- **`markdown_to_docx(markdown_text: str) -> Document`**（`app/export_docx.py`）：将 Markdown 字符串转为 python-docx Document 对象，支持标题（# ~ ######）、段落、无序/有序列表、**粗体**、*斜体*、表格、分割线等。
- **GET /api/tasks/{task_id}/download**：调用 `assemble_full_markdown` 获取 Markdown → 调用 `markdown_to_docx` 转为 DOCX → 以 `StreamingResponse` 返回文件流。若 params/framework/chapters 任一未完成，返回 400。
- **前端「下载 Word」按钮**：在 `chaptersStep?.status === 'completed'` 时展示，点击后通过 Blob 下载 DOCX。

### 技术选型

| 组件 | 用途 |
|------|------|
| `markdown` | 将 Markdown 转为 HTML |
| `beautifulsoup4` | 解析 HTML 结构 |
| `python-docx` | 构建并输出 Word 文档 |

### 文件变更

| 操作 | 文件 |
|------|------|
| 新增 | `backend/app/export_docx.py` |
| 修改 | `backend/requirements.txt`（新增 markdown、beautifulsoup4） |
| 修改 | `backend/app/config.py`（新增 EXPORT_DIR，可选） |
| 修改 | `backend/app/routers/tasks.py`（新增 download 路由） |
| 修改 | `frontend/src/api/tasks.ts`（新增 downloadTaskDocx） |
| 修改 | `frontend/src/pages/TaskDetailPage.tsx`（新增下载按钮与处理逻辑） |

**验收标准**（来自开发计划）：按章生成完成后，点击「下载 Word」可下载 DOCX；用 Word 打开，内容完整、结构清晰（项目信息 + 各章 + 风险点）。

### 步骤 6：通过 API 测试下载（可选）


### 步骤 7：通过页面端到端测试（推荐）

1. 打开 http://localhost:5173。
2. 进入一个**已完成按章生成**的任务（`chapters` 步骤状态为 `completed`）。
3. 在「步骤 5：按章生成」区块，应看到**「按章生成完成。」**文案后有一个**「下载 Word」**按钮。

打开下载的 `.docx` 文件，确认包含以下结构：

2. **项目信息**：`项目信息`（Heading 2）下为项目名称、规模等列表
3. **各章内容**：按框架章节顺序，每章有标题和正文
4. **附录**：`附录`（Heading 2）→ `风险点提醒`（Heading 3）→ 风险点列表

## 六、常见问题

| 现象 | 可能原因 | 处理办法 |
|------|----------|----------|
| `ModuleNotFoundError: No module named 'markdown'` | 未安装新依赖 | 执行 `pip install markdown beautifulsoup4` |
| 点击「下载 Word」报错「请先完成按章生成」 | 任务 chapters 步骤未 completed | 先完成按章生成 |
| 下载的是 JSON 而非 DOCX | 接口返回 4xx 错误 | 查看返回的 `detail` 字段，解决前置条件 |
| DOCX 打开是空白或乱码 | Markdown 内容为空或转换异常 | 用 `test_assembler.py <task_id>` 检查聚合结果 |
| 前端无「下载 Word」按钮 | `chaptersStep?.status` 不是 `completed` | 检查任务步骤状态，或刷新页面 |
| 下载文件名乱码 | 浏览器不支持 UTF-8 filename | 文件名已有 ASCII fallback，可正常使用 |
