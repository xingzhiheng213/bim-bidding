# 阶段 5.3 基础体验 — 操作与验证指南（新手向）

本文档面向新手，按「要达成什么 → 操作步骤 → 如何验证」说明阶段 5.3 的完整流程。阶段 5.3 为**基础体验**：统一加载态与成功/失败提示（Ant Design Message）；任务列表页展示历史任务、状态、创建时间，点击进入任务详情继续操作。

---

## 一、阶段 5.3 要达成什么

- **统一加载态、成功/失败提示**：各页使用 Ant Design Message（success/error/warning/loading）与 Spin/loading；首页创建任务成功显示「任务已创建」，失败用 message.error 展示，与任务详情、设置页一致。
- **任务列表页**：首页展示历史任务列表（任务 ID、状态、创建时间），支持行点击或「进入」链接进入任务详情；列表有加载态与空态（暂无任务时提示创建）。
- **验收标准**（来自开发计划）：统一加载态与成功/失败提示；任务列表页可展示历史任务并点击进入详情。

## 三、操作步骤（按顺序做）

### 步骤 1：确认服务已启动

按 [开发前-服务启动顺序.md](开发前-服务启动顺序.md) 依次启动：后端（uvicorn）、Celery Worker、前端（npm run dev）。浏览器可访问 http://localhost:5173。

---

### 步骤 2：验证首页任务列表

1. 打开 http://localhost:5173（首页）。
2. 确认页面有「任务列表」区块，表格列包含**任务 ID**、**状态**、**创建时间**、**操作**（进入）。
3. 若有任务：表格展示任务，创建时间格式为本地时间；点击某行或「进入」应跳转到该任务详情页（`/tasks/{id}`）。
4. 若无任务：表格显示「暂无任务，请点击上方按钮创建」。
5. 列表加载时表格显示 loading 态（或 Spin）。

---

### 步骤 3：验证创建任务的成功/失败提示

1. 在首页点击**「创建任务」**。
2. 成功时：应出现 **message.success「任务已创建」**，并自动跳转到新任务详情页。
3. 失败时（如关闭后端再点创建）：应出现 **message.error** 提示错误内容，与任务详情、设置页的错误提示方式一致；无内联红色文案即可。

---

### 步骤 4：验证列表与详情联动

1. 在首页创建新任务后跳转到详情，再通过顶部「首页」返回。
2. 确认任务列表中包含刚创建的任务（创建成功后已 invalidate 列表，返回时列表会刷新）。

---

## 四、验证（阶段 5.3 验收）

按下面三项逐项检查，都通过即表示阶段 5.3 完成。

### 验证 1：统一加载态、成功/失败提示

- 首页创建任务：成功有 message.success「任务已创建」；失败有 message.error，无仅内联错误文案。
- 与任务详情、设置页一致：各页主要操作使用 Message 与 loading/Spin。

### 验证 2：任务列表页展示历史任务

- 首页展示任务列表（任务 ID、状态、创建时间）；创建时间可读（本地时间格式）。
- 列表有加载态；无任务时显示空态文案（如「暂无任务，请点击上方按钮创建」）。

### 验证 3：点击进入任务详情

- 点击表格某行或「进入」链接，可跳转到对应任务详情页（`/tasks/{id}`），并正常查看/操作该任务。

## 五、常见问题

| 现象 | 可能原因 | 处理办法 |
|------|----------|----------|
| 任务列表为空且从未创建过任务 | 正常 | 显示「暂无任务，请点击上方按钮创建」；点击「创建任务」后跳转详情，返回首页可见新任务 |
| 列表一直 loading | 后端未启动或 GET /api/tasks 失败 | 确认后端已启动；打开开发者工具 Network 查看 /api/tasks 是否 200 |
| 点击行无跳转 | onRow 未绑定或路由错误 | 确认 HomePage 中 Table 的 onRow 调用了 navigate(\`/tasks/${record.id}\`)；或点击「进入」链接应使用 Link to |
| 创建成功无 Toast | message.success 未调用 | 确认 createMutation.onSuccess 中有 message.success('任务已创建') |
