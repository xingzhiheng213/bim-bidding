# 阶段 0.3 基础数据模型 — 操作与验证指南（新手向）

本文档面向新手，按「需要准备什么 → 一步步操作 → 如何验证」说明阶段 0.3 的完整流程。阶段 0.3 实现**基础数据模型**（任务表、步骤表）及前后端联调：能创建任务、能查到该任务及其步骤列表。

---

## 一、阶段 0.3 要达成什么

- 后端在 **PostgreSQL** 中有表 `tasks`、`task_steps`（后端启动时自动建表）。
- 能通过 **POST /api/tasks** 创建任务，创建时自动初始化若干步骤（如 upload、analyze、framework、chapters、export），状态均为 pending。
- 能通过 **GET /api/tasks/{id}** 获取任务详情及步骤列表。
- 前端首页有「创建任务」按钮，点击后创建任务并**跳转到任务详情页**，详情页展示该任务的 ID、状态、创建时间及**步骤列表**（步骤名、状态、创建时间等）。

**验收标准**（来自开发计划）：能创建一条任务，能查到该任务及其步骤列表；前后端能配合调用上述接口。

## 三、环境准备（0.3 无新增环境）

- **Redis、PostgreSQL**：与 0.1 相同，本机已安装并已能连上即可；0.3 不新增服务。
- **建表**：后端首次启动时会在数据库中执行 `create_all`，自动创建 `tasks`、`task_steps` 表，无需手动执行 SQL。
- **前端依赖**：0.3 增加了 `react-router-dom`，若尚未安装，在 `frontend` 目录下执行一次 `npm install` 即可。

## 四、操作步骤（按顺序做）

### 步骤 1：确认 Redis、PostgreSQL 已运行

按 [开发前-服务启动顺序.md](开发前-服务启动顺序.md) 的「二」检查：Redis（如 `redis-cli ping` 返回 PONG）、PostgreSQL（如 pgAdmin 能连上或服务已启动）。若未运行则先启动，再继续。

---

### 步骤 2：启动后端（第一个终端）

在终端（PowerShell 或 CMD）中执行：

```powershell
cd d:\标书工作流\backend
.venv\Scripts\activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

看到 `Uvicorn running on http://0.0.0.0:8000`、`Application startup complete.` 以及 `Tables created or already exist`（或类似）即表示后端就绪，表已建好。**此窗口不要关**。

---

### 步骤 3：启动前端（第二个终端）

再开一个终端，执行：

```powershell
cd d:\标书工作流\frontend
npm install
npm run dev
```

若之前已执行过 `npm install` 且已安装 `react-router-dom`，可只执行 `npm run dev`。看到 `Local: http://localhost:5173/` 即表示前端就绪。**此窗口不要关**。

---

### 步骤 4：浏览器验证（创建任务 + 任务详情）

1. 浏览器打开：<http://localhost:5173>
2. 页面应显示「BIM 标书生成」标题和「后端状态：ok」，以及**「创建任务」**按钮。
3. 点击「创建任务」：
   - 若成功：页面会**自动跳转**到任务详情页（地址类似 `http://localhost:5173/tasks/1`）。
4. 在任务详情页应看到：
   - **任务 ID**、**状态**（pending）、**创建时间**；
   - **步骤列表**表格：包含多行，每行有「步骤」（如 upload、analyze、framework、chapters、export）、「状态」（pending）、「创建时间」。

若点击「创建任务」后报错或未跳转：检查后端终端是否有报错；确认后端 8000 端口可访问（如浏览器打开 http://localhost:8000/health 得到 `{"status":"ok"}`）。

## 五、验证（阶段 0.3 验收）

按下面三项逐项检查，都通过即表示阶段 0.3 完成。

### 验证 1：后端表与接口

1. **表已存在**：用 pgAdmin 或 psql 连上本机 PostgreSQL、数据库 `bim_bidding`，应能看到表 `tasks`、`task_steps`，且字段与开发计划一致（tasks：id, user_id, status, created_at, updated_at；task_steps：id, task_id, step_key, status, input_snapshot, output_snapshot, error_message, created_at, updated_at）。
2. **POST 创建任务**：用浏览器或 Postman 请求 `POST http://localhost:8000/api/tasks`，body 可为空 `{}` 或 `{"initial_steps": ["upload", "analyze"]}`，应返回 201，body 含 `id`、`status`、`created_at`。
3. **GET 任务详情**：请求 `GET http://localhost:8000/api/tasks/1`（1 换成上一步返回的 id），应返回 200，body 含任务字段及 `steps` 数组，steps 中每项含 `step_key`、`status` 等。

---

### 验证 2：前端能创建任务并跳转

1. 保持后端、前端都在运行，浏览器打开 http://localhost:5173 。
2. 点击「创建任务」按钮。
3. 应无报错，且页面**跳转**到任务详情页（URL 为 `/tasks/数字`）。

若按钮一直转圈或报「后端请求失败」：确认后端 8000 端口已启动；检查前端 `.env.development` 中 `VITE_API_BASE=http://localhost:8000`。

---

### 验证 3：任务详情页展示任务与步骤列表

1. 在任务详情页（或通过「创建任务」跳转进入）。
2. 页面应展示：
   - 任务 **ID**、**状态**、**创建时间**；
   - **步骤列表**表格：多行数据，包含步骤名（step_key）、状态（status）、创建时间（created_at）。

若步骤列表为空：检查该任务是否在创建时初始化了步骤（默认应有 upload、analyze、framework、chapters、export 五步）；可用 GET /api/tasks/{id} 直接看后端返回是否含 steps。

## 六、常见问题

| 现象 | 可能原因 | 处理办法 |
|------|----------|----------|
| 点击「创建任务」后报错或未跳转 | 后端未启动或端口不对 | 先启动 uvicorn（端口 8000）；浏览器访问 http://localhost:8000/health 确认 |
| 前端报错 Failed to resolve "react-router-dom" | 未安装前端依赖 | 在 `frontend` 下执行 `npm install`，再执行 `npm run dev` |
| 任务详情页步骤列表为空 | 创建任务时未初始化步骤或接口异常 | 用 GET /api/tasks/1 看返回；确认后端 `POST /api/tasks` 会创建默认 5 个步骤 |
| 后端启动报错找不到表 | 数据库连接失败或未建表 | 确认 PostgreSQL 已启动、`.env` 中 `DATABASE_URL` 正确；重启 uvicorn 会再次执行 create_all |
| GET /api/tasks/{id} 返回 404 | 任务 id 不存在 | 先用 POST 创建任务，再用返回的 id 访问详情 |
