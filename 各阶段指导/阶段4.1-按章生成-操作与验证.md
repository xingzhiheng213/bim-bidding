# 阶段 4.1 按章生成 — 操作与验证指南（新手向）

本文档面向新手，按「需要准备什么 → 一步步操作 → 如何验证」说明阶段 4.1 的完整流程。阶段 4.1 实现**按章生成**：在框架步骤已确认（status=completed）后，可根据框架章节选择「全部」或「指定章节」触发按章生成；后端对每一章顺序执行「小节大纲 LLM → 知识库检索 → 章节正文 LLM」，结果写入 chapters 步骤的 output_snapshot；前端支持章节多选、进度展示（正在生成第 X 章 / 共 Y 章）、完成后可展开预览每章正文。

---

## 一、阶段 4.1 要达成什么

- **后端接口**：**POST /api/tasks/{task_id}/steps/chapters/run**（返回 202）。仅当 framework 步骤 status 为 **completed** 且 framework.output_snapshot.chapters 非空时允许调用。Body 可选：`{ "chapter_numbers": [1, 2, 3] }`，缺省或空数组表示全部章节。若 chapters 步骤已在 running 则返回 202 且 message「按章生成已在进行中」；否则将 chapters 步骤置为 running、写入初始 output_snapshot（total、current=0、chapters={}），入队 Celery 任务 **run_chapters**，返回 202「按章生成已入队」。
- **Celery 任务 run_chapters**：读 framework/analyze/params 步骤输出；对选中的每一章顺序执行：小节大纲 LLM → 知识库检索（query=章节 full_name）→ 章节正文 LLM；每章开始前将 output_snapshot.current 设为该章序号（便于前端显示「正在生成第 X 章」），每章完成后写入 output_snapshot.chapters[str(num)] 并提交；全部成功后 chapters 步骤 status=completed，任一章失败则 status=failed 并记录 error_message。
- **chapters 步骤 output_snapshot**：进行中为 `{ "total": N, "current": k, "chapters": { "1": "...", "2": "..." } }`，其中 current 为当前正在生成的章节号；完成后为 `{ "total": N, "chapters": { "1": "...", ... } }`。
- **前端**：当 framework 为 **completed** 时，在「步骤 5：按章生成」区块展示**选择要生成的章节**（Checkbox.Group，默认全选）、「全选」「取消全选」、**「开始按章生成」**按钮；点击后调用 chapters/run（可带 chapter_numbers）。chapters 步骤为 **running** 时轮询并显示「正在生成第 X 章 / 共 Y 章」（X 为实际章节号）；为 **completed** 时显示「按章生成完成」及可折叠的**章节预览**（每章可展开查看正文内容）。失败时展示错误与「重试」按钮。

**验收标准**（来自开发计划）：框架为 completed 时可调用 POST .../steps/chapters/run 入队按章生成；支持选择全部或指定章节；对每一章顺序执行小节大纲 LLM → 检索 → 章节正文 LLM，结果写入 chapters 步骤 output；前端能选择章节、展示进度（含选择单章时显示正确章节号如「正在生成第 5 章 / 共 1 章」）、完成后可预览每章正文。

## 三、环境准备

- **无新增依赖**：后端、前端沿用现有依赖；章节生成使用 **CHAPTER_LLM_PROVIDER**、**CHAPTER_LLM_MODEL**（默认可与 framework 一致，见 `.env.example`）。
- **服务顺序**：需要 **3 个终端**：后端（uvicorn）、Celery Worker、前端（npm run dev）。详见 [开发前-服务启动顺序.md](开发前-服务启动顺序.md)。
- **修改任务或 prompt 后**：需**重启 Celery Worker** 后新逻辑才会生效。

## 四、操作步骤（按顺序做）

### 步骤 1～4：与阶段 3.2 相同

确认 Redis、PostgreSQL 已运行；启动后端（终端 1）、Celery Worker（终端 2）、前端（终端 3）。

---

### 步骤 5：进入「框架已确认」的任务

1. 浏览器打开 <http://localhost:5173>，进入一个**框架已接受并继续**（framework 步骤 status=**completed**）的任务详情页。
2. 确认「步骤 4：框架生成」区块显示**「框架已确认，可进入按章生成。」**，且下方出现**「步骤 5：按章生成」**区块。

---

### 步骤 6：选择章节并开始按章生成

1. 在「步骤 5：按章生成」中，应看到**「选择要生成的章节：」**及若干复选框（第 1 章 xxx、第 2 章 xxx…），默认**全选**。
2. 可点击**「取消全选」**再勾选部分章节（如仅勾选「第 5 章 xxx」），或点击**「全选」**恢复全部。
3. 点击**「开始按章生成」**，应出现「按章生成已入队」类提示；「步骤 5」区块随即显示**「正在生成第 X 章 / 共 Y 章…」**（若只选了一章，X 应为该章序号，例如只选第 5 章时显示「正在生成第 5 章 / 共 1 章…」）。
4. 等待轮询更新，直至文案变为**「按章生成完成。」**（或失败时显示错误与「重试」）。

---

### 步骤 7：验证章节预览

1. 在按章生成**完成**后，应看到**「已生成 N 章，可展开预览：」**及一个可折叠列表。
2. 列表中每项为**「第 N 章」**，展开后为该章的**正文内容**（Markdown 原文，可滚动查看）。
3. 步骤列表中 chapters 步骤的**状态**应为 **completed**；GET /api/tasks/{id} 中 chapters 步骤的 output_snapshot 应含 `chapters` 对象，键为章节号、值为该章正文。

---

### 步骤 8：（可选）用接口验证 chapters/run

1. 对 framework 为 **completed**、chapters 为 pending 或 failed 的任务：**POST** /api/tasks/{id}/steps/chapters/run，body：`{}` 或 `{"chapter_numbers": [1, 3]}`。应返回 **202**，body 含 `"message": "按章生成已入队"`、`"step_key": "chapters"`。
2. 对 chapters 已为 **running** 的任务再次调用同一接口，应返回 **202** 且 message「按章生成已在进行中」。
3. 对 framework 未 completed 或无章节列表的任务调用，应返回 **400**。

---

## 五、验证（阶段 4.1 验收）

按下面五项逐项检查，都通过即表示阶段 4.1 完成。

### 验证 1：framework 为 completed 时可触发按章生成

1. 在 framework 步骤为 **completed** 时，调用 **POST /api/tasks/{id}/steps/chapters/run**（body 可为 `{}`）。
2. 应返回 **202**，chapters 步骤应先变为 **running**，由 Celery 执行完成后变为 **completed** 或 **failed**。

---

### 验证 2：支持选择全部或指定章节，且进度显示正确章节号

1. 在「步骤 5」中仅勾选**一章**（如第 5 章），点击「开始按章生成」。
2. 进度应显示**「正在生成第 5 章 / 共 1 章…」**（而非「正在生成第 1 章」）。
3. 选择多章或全选时，进度中的「第 X 章」应为当前正在生成的实际章节号。

---

### 验证 3：每章执行小节大纲 → 检索 → 章节正文，结果写入 output

1. 通过步骤列表或 GET 任务详情确认 chapters 步骤 **output_snapshot** 含 `chapters` 对象，键为章节号、值为该章 Markdown 正文。
2. 可选：查看 Celery 日志确认每章依次执行大纲 LLM、检索、正文 LLM。

---

### 验证 4：前端轮询展示进度与完成态

1. 触发按章生成后，页面应自动轮询，先显示「正在生成第 X 章 / 共 Y 章…」，完成后显示「按章生成完成。」。
2. 完成态下应出现「已生成 N 章，可展开预览：」及按章可折叠的预览列表。

---

### 验证 5：完成后可展开预览每章正文

1. 在「按章生成完成」后，展开任意「第 N 章」折叠项，应能看到该章的**正文内容**（可滚动，格式为 Markdown 原文）。
2. 章节号顺序与框架一致，内容与 output_snapshot.chapters 中对应键一致。

## 六、常见问题

| 现象 | 可能原因 | 处理办法 |
|------|----------|----------|
| 点击「开始按章生成」后报 400 | framework 未 completed 或框架无章节列表 | 先完成「接受并继续」，确认框架步骤为 completed 且章节列表非空 |
| 进度一直显示「正在生成…（共 N 章）」不出现章节号 | 后端未在每章开始前写入 current | 确认 `backend\tasks\chapters.py` 中在每章循环开头有写入 `out["current"] = num` 并 commit；重启 Celery |
| 只选一章却显示「正在生成第 1 章」 | 前端用「第几个选中项」而非实际章节号 | 前端应显示 output_snapshot.current（实际章节号）；确认后端在**开始**生成本章时写入 current=该章序号 |
| 按章生成无反应或未入队 | Celery Worker 未开或未加载 tasks.chapters | 开终端在 backend 下执行 `celery -A celery_app worker -l info -P solo`；确认 `celery_app\__init__.py` 的 include 含 `tasks.chapters` |
| 完成后无章节预览 | 前端未解析 output_snapshot.chapters 或未渲染折叠 | 确认 TaskDetailPage 中 chaptersContent 来自 output_snapshot.chapters，且 completed 时渲染 Collapse 每章一项、children 为正文 |
| 某章生成失败 | LLM 超时、知识库异常或 prompt 导致 | 查看 chapters 步骤 error_message 与 Celery 日志；可调整 CHAPTER_LLM_*、LLM_TIMEOUT_READ 或重试 |
