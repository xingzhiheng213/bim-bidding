# 阶段 1.3 前端上传与状态 — 操作与验证指南（新手向）

本文档面向新手，按「需要准备什么 → 一步步操作 → 如何验证」说明阶段 1.3 的完整流程。阶段 1.3 实现**前端上传与状态**：在任务详情页提供文件选择与上传按钮，上传成功后自动触发「开始解析」，轮询任务状态，解析完成后展示「解析完成，共 xxx 字」及文本预览（后端 1.1、1.2 已就绪，本阶段仅改前端）。

---

## 一、阶段 1.3 要达成什么

- 任务详情页有**上传区块**：文件选择（.pdf/.doc/.docx）+「上传」按钮，调用 `POST /api/tasks/{task_id}/upload`；上传成功后提示「上传成功」，并**自动**调用 `POST .../steps/extract/run` 触发解析。
- **轮询**：当 extract 步骤为 running 时，前端每约 2 秒请求一次 `GET /api/tasks/{id}`，直到 extract 变为 completed 或 failed。
- **解析结果展示**：extract 完成时展示「解析完成，共 xxx 字」及可折叠的「预览」（前几百字）；extract 失败时展示错误信息。
- 验收时**在网页上**完成：首页创建任务 → 进入任务详情 → 选择 PDF 或 DOCX 上传 → 看到上传成功、解析中、解析完成及字数与预览，且无报错。

**验收标准**（来自开发计划）：上传一个 PDF 或 DOCX 后，能在任务中看到解析出的文本内容（或字数），且无报错。

## 四、操作步骤（按顺序做）

### 步骤 1：确认后端、Celery、前端已运行

按 [开发前-服务启动顺序.md](开发前-服务启动顺序.md) 确认：

- **终端 1**：backend 下已执行 `uvicorn app.main:app --reload --host 0.0.0.0 --port 8001`，看到 `Application startup complete.`
- **终端 2**：backend 下已执行 `celery -A celery_app worker -l info -P solo`，看到 `celery@xxx ready.`
- **终端 3**：frontend 下已执行 `npm run dev`，看到 `Local: http://localhost:5173/`

若任一未运行，先按该文档启动，再继续。

---

### 步骤 2：浏览器打开前端

1. 浏览器打开：<http://localhost:5173>
2. 应看到「BIM 标书生成」首页，有「创建任务」按钮。

---

### 步骤 3：创建任务并进入任务详情

1. 点击首页的**「创建任务」**。
2. 页面应**自动跳转**到任务详情页（地址类似 `http://localhost:5173/tasks/6`）。
3. 任务详情页应显示：任务 ID、状态、创建时间，以及**「上传招标文件」**区块（文件选择 +「上传」按钮）和**步骤列表**表格。

---

### 步骤 4：选择文件并上传

1. 在「上传招标文件」区块，点击**选择文件**，选一个本机的 **PDF 或 DOCX**（如「测试文档.pdf」）。
2. 点击**「上传」**按钮。
3. 应出现**「上传成功」**的提示（页面顶部或中部短提示）。
4. 上传成功后，**无需再点「开始解析」**：前端会自动触发解析；步骤列表中会出现或更新 **extract** 步骤，状态先为 **running**。

---

### 步骤 5：等待解析完成并查看结果

1. 页面在解析期间会显示**「解析中，请稍候…」**（或步骤表中 extract 为 running）。
2. 前端会**自动轮询**任务详情（约每 2 秒一次），无需手动刷新。
3. 解析完成后（通常几秒到十几秒，视文件大小而定），应看到：
   - **「解析完成，共 xxx 字。」**（xxx 为提取出的纯文本字数）
   - **「已上传：xxx.pdf」**（或你上传的文件名）
   - 可折叠的**「预览」**区块，展开后可看到提取出的前几百字正文。
4. 步骤列表中 **extract** 步骤状态应为 **completed**。

若解析失败（如上传了 .doc 或文件损坏），会显示红色**「解析失败」**及具体错误信息（如「暂不支持 .doc」）。

---

### 步骤 6：（可选）确认步骤列表

在任务详情页底部**步骤列表**表格中，应能看到多行步骤（如 upload、analyze、framework、chapters、export、extract 等），每行有步骤名、状态、创建时间。upload 与 extract 的状态应与上方展示一致。

---

## 五、验证（阶段 1.3 验收）

按下面三项逐项检查，都通过即表示阶段 1.3 完成。

### 验证 1：上传与自动触发解析

1. 在任务详情页选择 PDF 或 DOCX，点击「上传」。
2. 应出现「上传成功」提示，且**未**再点击「开始解析」时，页面即进入解析状态（出现「解析中」或步骤表中 extract 为 running）。
3. 若后端或 Celery 未开，上传可能成功但解析一直不完成，此时需检查终端 1、2 是否在跑。

---

### 验证 2：解析完成后展示字数与预览

1. 等待解析完成（extract 步骤变为 completed）。
2. 页面应展示**「解析完成，共 xxx 字。」**，且 xxx 为大于 0 的整数（与文件内容一致）。
3. 展开**「预览」**区块，应能看到从 PDF/DOCX 中提取出的前几百字正文，内容合理、无乱码。

---

### 验证 3：失败时展示错误信息

1. 可选：新建一个任务，上传一个 **.doc** 文件（或对未上传文件的任务通过接口直接调 extract/run）。
2. 解析失败时，页面应展示**「解析失败」**及具体 **error_message**（如「暂不支持 .doc，请使用 .docx 或 PDF」），且步骤列表中 extract 为 **failed**。

## 六、常见问题

| 现象 | 可能原因 | 处理办法 |
|------|----------|----------|
| 点击「上传」后无反应或报错 | 后端未启动或端口不对 | 确认终端 1 在跑 uvicorn（端口 8001）；浏览器访问 http://localhost:8001/health 应得到 `{"status":"ok"}` |
| 上传成功但一直「解析中」 | Celery Worker 未开 | 确认终端 2 在跑 `celery -A celery_app worker -l info -P solo`；见 [阶段1.2-文档解析-操作与验证.md](阶段1.2-文档解析-操作与验证.md) |
| 页面没有「上传招标文件」区块 | 该任务 upload 步骤已是 completed | 1.3 仅在上传未完成时显示上传区；可回到首页「创建任务」新建一条任务再测 |
| 修改前端代码后页面没变 | 浏览器缓存或 Vite 未热更新 | 刷新页面（F5 或 Ctrl+R）；若仍无变化，确认 frontend 终端在跑 `npm run dev` |
| 预览里是乱码或空 | 文件本身无文字或解析异常 | 换一个含正文的 PDF/DOCX 再试；若仍异常，看步骤列表中 extract 的 error_message |
