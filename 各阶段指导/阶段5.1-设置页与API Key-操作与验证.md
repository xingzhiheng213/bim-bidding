# 阶段 5.1 设置页与 API Key — 操作与验证指南（新手向）

本文档面向新手，按「需要准备什么 → 一步步操作 → 如何验证」说明阶段 5.1 的完整流程。阶段 5.1 为**设置页与 API Key**：在 App 设置中配置大模型 API Key 与 Base URL；支持按步骤选择模型（分析、参数提取、框架生成、按章生成）；密钥加密存入 DB，任务执行时 Celery 从设置读取，未配置时提示「请在设置中配置 xxx API Key」。

---

## 一、阶段 5.1 要达成什么

- **大模型 API**：为 DeepSeek、智谱提供 API Key 输入框（密码框）和可选 Base URL；保存后加密写入 DB，仅返回脱敏 key（如 `sk-***xxx`）和配置状态。
- **模型配置**：可选择「默认模型」，以及各步骤（分析、参数提取、框架生成、按章生成）单独使用的模型；模型名对应供应商（如 glm-4-flash → 智谱），需先在「大模型 API」中配置对应 API Key。
- **任务执行**：Celery 各步骤通过 `get_llm_for_step(step_key)` 解析出 (provider, model)，再从设置/DB 读取 API Key 与 Base URL；若缺失则步骤失败并提示「请在设置中配置 xxx API Key」。
- **验收标准**（来自开发计划）：在设置中填写并保存 API Key 后，新建任务能正常调用 LLM；未配置时提示明确；可为不同步骤选择不同模型。

---

## 二、你需要提前准备的东西

| 项目 | 说明 | 如何确认 |
|------|------|----------|
| 阶段 0–4 已可用 | 后端、Celery、前端、任务流程可跑通 | 能创建任务、上传、解析、分析、框架、按章生成（或至少到分析步骤） |

| 项目代码 | 设置页、设置 API、模型解析与加密存储已实现 | 存在 `backend/app/routers/settings.py`、`backend/app/settings_store.py`、`frontend/src/pages/SettingsPage.tsx` |

## 三、环境准备

- **后端**：可选在 `.env` 中设置 `SETTINGS_SECRET_KEY`（32 字节 base64，用于加密 API Key）；**未设置时**使用固定开发占位密钥，FastAPI 与 Celery 共用，便于本地联调。
- **服务顺序**：需要 **3 个终端**：后端（uvicorn）、Celery Worker、前端（npm run dev）。详见 [开发前-服务启动顺序.md](开发前-服务启动顺序.md)。修改设置后若 Celery 已启动，无需重启即可读取 DB；若曾修改 `SETTINGS_SECRET_KEY` 或加密逻辑，需重启后端与 Celery。

---

## 四、操作步骤（按顺序做）

### 步骤 1：确认 Redis、PostgreSQL 已运行

- Redis：`redis-cli ping` 返回 `PONG`。
- PostgreSQL：服务运行中；若之前报错「数据库结构可能过旧」，需对 `llm_settings` 执行 `ALTER TABLE llm_settings ADD COLUMN base_url VARCHAR(512);`（见第六节常见问题）。

---

### 步骤 2：启动后端、Celery、前端

按 [开发前-服务启动顺序.md](开发前-服务启动顺序.md) 依次启动：终端 1 后端、终端 2 Celery Worker、终端 3 前端。浏览器可访问 http://localhost:5173。

---

### 步骤 3：打开设置页并配置大模型 API

1. 浏览器打开 http://localhost:5173，点击顶部「设置」进入设置页。
2. 在「大模型 API」区块，至少为一个供应商（如智谱）填写 **API Key**；**Base URL** 可选，留空使用默认（智谱默认 `https://open.bigmodel.cn/api/paas/v4`）。
3. 点击该供应商旁的**「保存」**，应出现「已保存」提示；下方显示「已配置」及脱敏 key（如 `sk-***xyz`）。

---

### 步骤 4：配置模型（默认模型与各步骤模型）

1. 在「模型配置」区块，选择**默认模型**（如 `deepseek-chat (deepseek)` 或 `glm-4-flash (zhipu)`）。
2. 可为「分析」「参数提取」「框架生成」「按章生成」分别选择模型；未单独选择的步骤使用默认模型。例如将「分析」选为智谱某模型，则分析步骤会使用智谱 API，需已保存智谱 API Key。
3. 选择后会自动保存（或按界面说明点击保存）；保存成功会有提示。

---

### 步骤 5：验证任务执行是否使用设置中的 Key 与模型

1. 在首页创建新任务或进入已有任务，完成「解析」后触发**分析**步骤（若分析步骤已配置为智谱，则需已保存智谱 API Key）。
2. 确认分析步骤能正常完成，无「请在设置中配置 zhipu API Key」等错误。
3. 若故意不配置某供应商的 API Key 却将某步骤选为该供应商的模型，该步骤应失败并提示「请在设置中配置 xxx API Key」。

---

## 五、验证（阶段 5.1 验收）

按下面四项逐项检查，都通过即表示阶段 5.1 完成。

### 验证 1：设置页可保存 API Key 与 Base URL

1. 打开设置页，为 DeepSeek 或智谱填写 API Key，点击保存，出现「已保存」。
2. 页面显示「已配置」及脱敏 key；再次打开设置页，仍显示已配置（不显示明文）。
3. 可选：填写 Base URL 并保存，或留空使用默认；保存后无报错。

---

### 验证 2：模型配置可保存并生效

1. 在「模型配置」中选择默认模型；为「分析」等步骤单独选择模型（如分析用智谱）。
2. 保存后无报错；刷新页面后选择仍保持。
3. 新建任务并执行到分析步骤时，分析步骤使用的模型与设置一致（且需已配置对应 API Key）。

---

### 验证 3：未配置 API Key 时提示明确

1. 将某步骤（如分析）选为某供应商的模型，但不在「大模型 API」中保存该供应商的 API Key（或清空后不保存）。
2. 触发该步骤，应失败并显示「请在设置中配置 xxx API Key」（或类似明确提示）。

---

### 验证 4：Celery 任务能读取设置中的 Key（加密一致）

1. 在设置中保存智谱 API Key 后，不重启 Celery，直接触发使用智谱的分析步骤。
2. 分析步骤应成功完成（说明 Celery 从 DB 正确解密并读取 API Key）。若曾出现「请在设置中配置 zhipu API Key」且确认已保存，多为加密密钥不一致：未设置 `SETTINGS_SECRET_KEY` 时，需**重启后端与 Celery** 后**在设置页重新保存一次 API Key**，使加密与固定开发占位密钥一致。

---

## 六、常见问题

| 现象 | 可能原因 | 处理办法 |
|------|----------|----------|
| 点击保存显示「Network Error」 | 后端未启动或前端请求地址错误 | 确认后端已启动（如 http://localhost:8001）；检查前端 `VITE_API_BASE`（如 `.env.development`） |
| 保存 API Key 时报「数据库结构可能过旧」 | `llm_settings` 表缺少 `base_url` 列 | 在 PostgreSQL 中执行：`ALTER TABLE llm_settings ADD COLUMN base_url VARCHAR(512);`（见开发计划 / 阶段说明） |
| 分析（或其它步骤）报「请在设置中配置 zhipu API Key」但设置里已保存 | 未设 `SETTINGS_SECRET_KEY` 时，FastAPI 与 Celery 曾用不同随机密钥加解密 | 重启后端与 Celery 后，到设置页**重新输入并保存**一次该供应商的 API Key，再重试该步骤 |
| 启动后端报「Fernet key must be 32 url-safe base64-encoded bytes」 | 开发占位密钥长度曾写错 | 已修复为 32 字节；更新代码后重启即可 |
| 模型下拉为空或无法保存模型配置 | 后端 `/api/settings/models` 未返回或写入失败 | 检查后端日志；确认 `platform_llm_config` 表存在（由 Base.metadata.create_all 创建） |
