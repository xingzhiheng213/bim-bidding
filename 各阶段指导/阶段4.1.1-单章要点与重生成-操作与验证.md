# 阶段 4.1.1 单章要点与重生成 — 操作与验证指南（新手向）

本文档面向新手，按「需要准备什么 → 一步步操作 → 如何验证」说明阶段 4.1.1 的完整流程。阶段 4.1.1 实现**单章要点与重生成**：在按章生成完成后，对**已生成的单章**支持用户提交修改建议（要点），并支持「重新生成本章」仅重跑该章、按要点更新该章正文，其余章节不变。思路与框架层「添加要点 → 重新生成」一致。

---

## 一、阶段 4.1.1 要达成什么

### 思路（与框架层一致）

「添加要点」是对**当前章节正文的修改建议**。用户保存要点后，应点击「重新生成本章」，由 LLM 根据「当前正文 + 用户要点」更新该章；未提及的部分必须原样保留。满意后该章即更新完成。

### 后端

- **POST /api/tasks/{task_id}/steps/chapters/save-points**
  - Body：`{ "chapter_number": 3, "added_points": ["要点1", "要点2"] }`。
  - 仅当 chapters 为 **completed** 且该章已存在于 `chapters` 时允许：将 `added_points` 写入 `output_snapshot.chapter_points[str(chapter_number)]`，**不改变步骤状态**。
- **POST /api/tasks/{task_id}/steps/chapters/regenerate**
  - Body：`{ "chapter_number": 3 }`，可选 `{ "chapter_number": 3, "added_points": ["要点1"] }`。
  - 仅当 chapters 为 **completed** 且该章已存在时允许。若 body 带 `added_points` 则合并到 `chapter_points`；将 chapters 置为 **running**、`current` 设为该章号；入队 Celery 任务 **regenerate_chapter**，返回 202「该章已重新入队」。
  - 若 chapters 已为 **running**，返回 202 且 message「按章生成已在进行中」。
- **Celery 任务 regenerate_chapter**：
  - 若 `chapter_points` 存在：构建 prompt「当前正文 + 用户要点」，调用 LLM 生成新正文，使用较低 temperature（0.2）。
  - 若无要点：沿用完整流程（小节大纲 LLM → 检索 → 章节正文 LLM）。
  - 将新正文写回 `chapters[str(chapter_number)]`，其余章不变；`status` 置为 **completed**，移除 `current`。

### 前端

- 在「步骤 5：按章生成」**完成**后的**章节预览**处，每章提供「添加要点」「重新生成本章」按钮。
- 「添加要点」：弹窗输入多行要点，提交调用 **save-points**；成功后提示「已保存要点，可点击『重新生成本章』使该章按您的要点更新」。
- 「重新生成本章」：调用 **regenerate**，入队后轮询；chapters 步骤显示「正在生成第 X 章 / 共 Y 章…」；完成后该章正文更新。

**验收标准**：对已生成的某一章可保存要点；可触发「重新生成本章」且仅该章更新、其余章不变；前端在预览处可操作并看到更新结果。

## 四、操作步骤（按顺序做）

### 步骤 1～4：与阶段 4.1 相同

确认 Redis、PostgreSQL 已运行；启动后端（终端 1）、Celery Worker（终端 2）、前端（终端 3）。

---

### 步骤 5：进入「按章生成已完成」的任务

1. 浏览器打开 <http://localhost:5173>，进入一个**按章生成已完成**（chapters 步骤 status=**completed**）的任务详情页。
2. 确认「步骤 5：按章生成」区块显示**「按章生成完成。」**，下方有**章节预览**（可折叠列表，每章可展开查看正文）。

---

### 步骤 6：为某章添加要点并保存

1. 展开任意一章（如第 3 章），应看到**「添加要点」**和**「重新生成本章」**按钮。
2. 点击**「添加要点」**，弹窗标题应为「为第 3 章添加要点」。
3. 在文本框中输入若干要点（每行一个），例如：
   - 增加对 LOD 300 的说明
   - 补充碰撞检测流程
4. 点击**「保存要点」**，应出现提示「已保存要点，可点击『重新生成本章』使该章按您的要点更新」。
5. 关闭弹窗后，该章区域应显示**「已添加 N 个要点」**（N 为输入的要点数）。

---

### 步骤 7：重新生成本章

1. 点击**「重新生成本章」**，应出现「该章已重新入队」类提示。
2. 「步骤 5」区块随即显示**「正在生成第 3 章 / 共 N 章…」**（N 为总章数）。
3. 等待轮询更新，直至文案恢复为**「按章生成完成。」**。
4. 展开该章，正文应已按要点更新；**其余章节**应保持不变。

---

### 步骤 8：（可选）用接口验证 save-points 和 regenerate

1. **save-points**：对 chapters 为 **completed** 的任务，**POST** /api/tasks/{id}/steps/chapters/save-points，body：`{ "chapter_number": 3, "added_points": ["要点A", "要点B"] }`。应返回 **200**，body 含 `"message": "已保存要点"`。GET /api/tasks/{id} 中 chapters 步骤的 output_snapshot.chapter_points["3"] 应为 `["要点A", "要点B"]`。
2. **regenerate**：**POST** /api/tasks/{id}/steps/chapters/regenerate，body：`{ "chapter_number": 3 }`。应返回 **202**，body 含 `"message": "该章已重新入队"`。chapters 步骤 status 应先变为 **running**，current 为 3；Celery 完成后 status 恢复 **completed**，该章正文更新。
3. 对 chapters 已为 **running** 的任务再次调用 regenerate，应返回 **202** 且 message「按章生成已在进行中」。
4. 对 chapters 未 completed 或该章不存在的任务调用，应返回 **400**。

## 五、验证（阶段 4.1.1 验收）

按下面三项逐项检查，都通过即表示阶段 4.1.1 完成。

### 验证 1：对已生成的某一章可保存要点

1. 在章节预览处点击某章的「添加要点」，输入要点并保存。
2. 应提示「已保存要点，可点击『重新生成本章』使该章按您的要点更新」。
3. GET /api/tasks/{id} 中 chapters.output_snapshot.chapter_points 应含该章号及对应要点数组。

---

### 验证 2：可触发「重新生成本章」且仅该章更新、其余章不变

1. 保存要点后，点击「重新生成本章」。
2. 应显示「正在生成第 X 章…」，完成后该章正文按要点更新。
3. 展开其他章节，正文应与重生成前一致，未被改动。

---

### 验证 3：前端在预览处可操作并看到更新结果

1. 每章预览处应有「添加要点」「重新生成本章」按钮。
2. 添加要点弹窗标题为「为第 X 章添加要点」，保存后显示「已添加 N 个要点」。
3. 重新生成本章后，该章正文更新，页面轮询展示最新内容。

## 六、常见问题

| 现象 | 可能原因 | 处理办法 |
|------|----------|----------|
| 点击「添加要点」后报 400 | chapters 未 completed 或该章未生成 | 先完成按章生成，确认该章在 chapters 中 |
| 点击「重新生成本章」后报 400 | chapters 未 completed 或该章不存在 | 同前 |
| 重新生成本章无反应或未入队 | Celery Worker 未开或未加载 regenerate_chapter | 开终端执行 `celery -A celery_app worker -l info -P solo`；确认 tasks.chapters 已注册 |
| 该章更新后其余章被改动 | 任务逻辑错误 | 确认 regenerate_chapter 只写回 `chapters[str(chapter_number)]`，不改动其他键 |
| 有要点但重生成未按要点更新 | chapter_points 未正确写入或未读取 | 确认 save-points 写入 chapter_points，regenerate_chapter 读取 chapter_points 构建 prompt |
