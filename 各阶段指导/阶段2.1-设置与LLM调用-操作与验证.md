# 阶段 2.1 设置与 LLM 调用 — 操作与验证指南（新手向）

本文档面向新手，按「需要准备什么 → 一步步操作 → 如何验证」说明阶段 2.1 的完整流程。阶段 2.1 实现**设置（环境变量）与 LLM 调用层**：在 `.env` 中配置 DeepSeek（或智谱）的 API Key 与 Base URL，封装 `call_llm` 通过 OpenAI 兼容接口请求大模型，供后续 2.2 分析步骤等使用。本阶段**不建 DB 设置表**，全部用环境变量；阶段 5 再做「设置页与 API Key 持久化」。

---

## 一、阶段 2.1 要达成什么

- **设置存储**：使用 **环境变量**（通过 `backend/.env` 配置），支持 `DEEPSEEK_API_KEY`、`ZHIPU_API_KEY` 及可选的 `DEEPSEEK_BASE_URL`、`ZHIPU_BASE_URL`；`app.config` 提供 `get_llm_api_key(provider)` 供调用方使用。
- **LLM 调用层**：`app/llm.py` 中的 `call_llm(provider, model, messages, temperature, ...)`，按 provider 取 key 与 base_url，用 httpx 请求 OpenAI 兼容的 `/chat/completions`，返回 `choices[0].message.content` 或抛错。
- **在 Celery 任务中**：后续 2.2 分析任务将根据 task 的 user_id 查设置取 api_key；本阶段**仅实现 fallback 到 env**（即统一用 `get_llm_api_key(provider)` 从 `.env` 取 key），不查 DB。
- 验收时通过 **脚本** `python test_llm.py` 调用 DeepSeek，能打出模型回复即表示 2.1 通过。

**验收标准**（来自开发计划）：配置好环境变量后，能通过脚本或简单任务调用 LLM 并拿到返回 content。

## 三、环境准备

- **无新增依赖**：2.1 使用项目已有的 `httpx`、`python-dotenv`，无需再执行 `pip install` 新包。
- **配置方式**：在 **backend 目录下** 的 **`.env`** 文件中添加 LLM 相关变量（**不要**去 Windows「环境变量」里添加）。若没有 `.env`，可复制 `.env.example` 为 `.env` 再编辑。
- **DeepSeek 示例**（先测 DeepSeek 即可）：
  - `DEEPSEEK_API_KEY=sk-你的密钥`
  - `DEEPSEEK_BASE_URL=https://api.deepseek.com`（可选；不写则使用代码默认 `https://api.deepseek.com/v1`）

## 四、操作步骤（按顺序做）

### 步骤 1：确认 backend 下有 .env 并配置 DeepSeek

1. 打开 **`d:\标书工作流\backend\.env`**（若不存在，将 `backend\.env.example` 复制为 `.env`）。
2. 在文件中添加或确认已有以下两行（等号两边无空格、无引号）：
   ```env
   DEEPSEEK_API_KEY=sk-你的DeepSeek密钥
   DEEPSEEK_BASE_URL=https://api.deepseek.com
   ```
3. 保存文件。  
说明：`DEEPSEEK_BASE_URL` 可不写，代码会使用默认值；若 DeepSeek 要求用根地址，则写 `https://api.deepseek.com`。

---

### 步骤 2：进入 backend 目录并激活虚拟环境

在 CMD 或 PowerShell 中执行：

```powershell
cd d:\标书工作流\backend
.venv\Scripts\activate
```

看到命令行前出现 **`(.venv)`** 即表示虚拟环境已激活。**必须**在虚拟环境下运行，否则会报 `ModuleNotFoundError: No module named 'httpx'` 等。

---

### 步骤 3：运行 LLM 验收脚本

在 **已激活 .venv 的** backend 目录下执行：

```powershell
python test_llm.py
```

预期输出类似：

- `Calling DeepSeek (deepseek-chat)...`
- `Response: （模型回复的一句话）`
- `OK`

若出现上述三行，即表示 2.1 通过；脚本会调用 `call_llm("deepseek", "deepseek-chat", [{"role": "user", "content": "你好，回复一句话"}])` 并打印返回内容。

---

### 步骤 4：（可选）智谱

若需同时测智谱，在 `.env` 中增加 `ZHIPU_API_KEY=...` 和（可选）`ZHIPU_BASE_URL=...`，并自行写一段调用 `call_llm("zhipu", "glm-4-flash", messages)` 的脚本或在后继分析步骤中使用。本阶段**仅要求 DeepSeek 验收通过**即可。

## 五、验证（阶段 2.1 验收）

按下面两项逐项检查，都通过即表示阶段 2.1 完成。

### 验证 1：.env 配置正确且被加载

1. `backend/.env` 中存在 **DEEPSEEK_API_KEY=sk-...**（且密钥有效）。
2. 运行 `python test_llm.py` 时**未**报错「未配置 deepseek API Key」或 `ValueError` 提示缺 key。

若报「未配置 xxx API Key」：检查 `.env` 是否在 **backend** 目录下、变量名是否为 **DEEPSEEK_API_KEY**、是否在等号前有多余空格或拼写错误。

---

### 验证 2：call_llm 能拿到模型回复

1. 在 backend 下、已激活 `.venv` 的情况下执行 `python test_llm.py`。
2. 终端应打印 **Response: （一句中文或英文回复）** 和 **OK**。
3. 若报 `httpx.HTTPStatusError` 或 401/403：多为 API Key 错误或未开通权限；检查 DeepSeek 控制台中该 key 是否有效、是否有余额或权限。

---

## 六、常见问题

| 现象 | 可能原因 | 处理办法 |
|------|----------|----------|
| ModuleNotFoundError: No module named 'httpx' 或 'app' | 未在虚拟环境中运行 | 先执行 `cd backend`、`.venv\Scripts\activate`，再运行 `python test_llm.py` |
| 未配置 deepseek API Key | .env 中无 DEEPSEEK_API_KEY 或未加载 | 确认 `.env` 在 **backend** 目录下；变量名 `DEEPSEEK_API_KEY`，等号两边无空格 |
| 401 Unauthorized / 403 Forbidden | API Key 错误或未开通 | 在 DeepSeek 开放平台检查 key 是否有效、是否已开通对应模型 |
| 连接超时或网络错误 | 网络或代理问题 | 检查本机网络、代理设置；必要时配置 HTTP_PROXY 等 |
| .env 改了但不生效 | 未保存或运行目录不对 | 保存 `.env` 后重新执行 `python test_llm.py`；脚本会切换 cwd 到 backend，一般无需改运行目录 |
